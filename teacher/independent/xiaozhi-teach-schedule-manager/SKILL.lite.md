# xiaozhi-teach-schedule-manager (Lite)

> 精简版 SKILL，完整定义与示例请参阅 `SKILL.md`。

## 触发条件

| 触发词 | 动作 |
|---|---|
| 排课 / 排下周的课 | 触发周课表生成 + 冲突检测 |
| 本周课表 / 下周课表 | 触发课表查看 |
| 调课 / 把 [化名] 的课调到 [日期] | 触发调课三步（双方确认+记录） |
| 补课 / 缺课要补 | 触发补课登记 + 排入空闲位 |
| 学员请假 / 请假怎么扣课时 | 触发请假登记 + 课时不扣规则 |
| 还剩几课时 / 课时包快到期 | 触发课时台账 + 续费预警 |
| 排课冲突 / 这个时间排了谁 | 触发冲突检测三类（老师/学员/课时包） |
| 下一节课是谁 | 触发下一节课查询 |
| 课时快用完了 | 触发课时预警（30%/10%/到期） |

## 核心流程骨架

```text
学员时间矩阵（来自 intake） + 老师时间矩阵
→ 周课表生成（固定学员优先+临时填空+留缓冲）
→ 冲突检测（老师时间/学员时间/课时包）
→ 排课建议输出（老师确认后写入）
→ 课时包台账管理（剩余/到期/续费预警）
→ 补课/请假/调课三动作（登记+留痕）
→ 写回 solo-dashboard
```

## 输出要点

- 学员时间矩阵（每周可用时间 + 不可用时间）
- 老师时间矩阵（每周可用课时位）
- 周课表生成三步走（固定优先/填空/留缓冲）
- 冲突检测三类（老师时间/学员时间/课时包预警）
- 补课/请假/调课三动作（标准流程+留痕）
- 课时包台账（剩余/到期/续费节点/预警）
- 续费预警三档（30%/10%/到期 7 天）
- 寒暑假/考试期/请假多特殊场景

## 禁止行为

| 禁止项 | 说明 |
|---|---|
| AI 替老师自动排课 | 排课建议老师确认后写入 |
| 擅自增加学员可上课时间 | 仅使用 intake 授权的偏好 |
| 课时台账不透明 | 实时更新剩余/到期/续费节点 |
| 超课时上课不预警 | 透支 1 课时允许，>1 暂停排课 |
| 公开课表暴露真实姓名 | 一律使用化名 |
| 调课不留记录 | 所有调课必须登记 |
| 微信沟通完就忘 | 补课/请假/调课必须有登记 |
