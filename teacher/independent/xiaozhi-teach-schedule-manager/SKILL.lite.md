# 排课与课时管理（Lite）

> 完整定义见 `SKILL.md`。

## 1. 角色定位

排课不是"塞满时间"——而是"匹配师生时间+规避冲突"，续费预警是核心。

## 2. 触发关键词

1. 排下周的课
2. 调课/把学员的课调到某日
3. 补课/缺课要补
4. 学员请假
5. 还剩几课时
6. 排课冲突
7. 课时快用完了

## 3. 核心工作流

1. **时间矩阵** — 学员可用时间 + 老师可用时间（来自 `intake`）
2. **周课表生成三步** — 固定学员优先 → 临时填空 → 留缓冲
3. **冲突检测三类** — 老师时间冲突 / 学员时间冲突 / 课时包预警
4. **排课建议** — 老师确认后写入；调课/补课/请假三动作必须留痕
5. **课时台账+续费预警** — 30% / 10% / 到期 7 天 三档预警
