# xiaozhi-teach-renewal-report (Lite)

> 精简版 SKILL，完整定义与示例请参阅 `SKILL.md`。

## 触发条件

| 触发词 | 动作 |
|---|---|
| 做一份阶段报告 | 触发中期/续费/期末报告模板 |
| 学员要续费了 | 触发续费报告 + 续费建议 |
| 续课报告怎么写 | 触发"事实+进步+计划"三段式 |
| 总结学员这学期 | 触发期末报告模板 |
| 家长要看学习报告 | 触发家长可读报告生成 |
| 续费话术 / 续费怎么跟家长说 | 触发续费沟通话术 |
| 课时过半 / 课时剩 30% | 触发中期/续费报告节点 |
| 家长犹豫续不续费 | 触发续费困难应对 |

## 核心流程骨架

```text
收集阶段数据（lesson-log/学情/课时）
→ 学员成长档案生成（5 维度+时间线）
→ 进步可视化（对比+热力图+轨迹）
→ 续课建议生成（课时包推荐+计划）
→ 续费沟通话术（→ parent-communication）
→ 老师确认后发送
```

## 输出要点

- 5 类触发节点（中期/续费/期末/节点/应急）
- 三段式报告结构（事实+进步+计划）
- 3 类报告模板（中期/续费/期末）
- 进步可视化（数据对比+热力图+时间线）
- 续费建议（课时包推荐逻辑+选项+优惠）
- 续费沟通话术（主话术+犹豫应对+困难处理）
- 数据来源与隐私边界

## 禁止行为

| 禁止项 | 说明 |
|---|---|
| AI 替老师自动发送报告 | 老师确认后通过 parent-communication 发送 |
| 编造或夸大进步数据 | 报告必须基于真实记录 |
| 承诺提分/排名/升学 | 只展示行为进步，不承诺分数 |
| 报告中暴露真实姓名 | 一律使用化名 |
| 强行挽留流失学员 | 体面结束，留下好印象 |
| 续费 = 强行推销 | 提供选项而非单一选项 |
| 贴长期标签 | "差生""笨"等标签禁用 |

## 协作声明

- 上游：`xiaozhi-teach-student-analyzer`（学情分析）
- 上游：`xiaozhi-teach-solo-dashboard`（课时/续费节点）
- 下游：`xiaozhi-teach-parent-communication`（续费沟通）
- 下游：`xiaozhi-teach-solo-dashboard`（续费记录）
