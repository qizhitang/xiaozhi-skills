> 配合 `xiaozhi-teach-renewal-report` 使用。

## 一、5 类触发节点

```text
┌──────────┬────────────┬──────────────┐
│ 节点      │ 触发条件     │ 形式          │
├──────────┼────────────┼──────────────┤
│ 中期报告  │ 课时包 50%  │ 阶段学习简报  │
│ 续费报告  │ 课时包 70%  │ 阶段成果+续费 │
│ 期末报告  │ 学期末/年度 │ 学期总结      │
│ 节点报告  │ 关键里程碑  │ 进步展示      │
│ 应急报告  │ 家长主动询问│ 即时简报      │
└──────────┴────────────┴──────────────┘
```

---

## 二、报告三段式结构

```text
■ 第一段：事实
  目的：客观陈述学习数据，不评价、不夸大
  内容：已上课时、学习主题、知识图谱、测评数据

■ 第二段：进步
  目的：基于事实展示真实进步
  内容：与起点对比、习惯养成、思维变化、抗挫能力

■ 第三段：计划
  目的：给出下阶段清晰学习路径
  内容：重点突破、课时建议、预期目标、续费建议
```

## 七、续费建议生成

### 7.1 课时包推荐逻辑

```text
■ 基于学习节奏
  当前每周 [N] 课时
  → 推荐课时包 = 每周 [N] 课时 × 目标周数 [N]

■ 基于学习目标
  短期目标（1 个月）：[N] 课时
  中期目标（3 个月）：[N] 课时
  长期目标（学期）：[N] 课时

■ 基于续费节点
  课时包 70% 触发续费
  续费提醒：[日期] 前完成
```

### 7.2 课时包选项

```text
■ 小包（10 课时）
  适合：短期试学 / 假期短期课
  时长：约 [X] 周
  特点：灵活、可调整

■ 中包（30 课时）
  适合：稳定期学员
  时长：约 [X] 周
  特点：性价比高、覆盖完整单元

■ 学期包（60+ 课时）
  适合：长期学员
  时长：约 [X] 周 / 整个学期
  特点：最优惠、稳定性最强
```

### 7.3 续费优惠

```text
■ 优惠类型
  · 早鸟优惠：到期前 14 天续费 [X] 折
  · 升级优惠：从小包升中包 [X] 折
  · 老学员优惠：连续 [N] 期续费 [X] 折
  · 推荐优惠：推荐新学员 [X] 优惠

⚠️ 优惠不是核心驱动
  续费核心是：学员真实进步 + 家长真实认可
  优惠只是"顺手"，不是"诱饵"
```

## 八、续费沟通话术

### 8.1 续费沟通主话术

```text
"X 妈/爸，您好！
[化名] 的 [X 课时包] 课时还剩 [N] 课时了。
我想跟您分享一下这段时间的学习情况，
也聊一下接下来的安排。

[插入本报告的"事实+进步+计划"三段]

针对 [化名] 当前的节奏和目标，
我推荐 [N 课时包]（约 [X] 周）：
  · 适合 [化名] 当前每周 [N] 课时的节奏
  · 可以完整覆盖 [X 阶段]
  · 续费优惠：[X 优惠信息]

如果您觉得需要调整课时或节奏，
随时跟我沟通，我们可以一起看怎么安排最好。
如果决定续费，回复 1 我来接龙。"
```

### 8.2 家长犹豫的应对

```text
"X 妈/爸，我理解您的考虑。
学习是个长期的事，节奏和方式都可以根据孩子情况调整。

您现在主要担心的是 [X 方面]？
  · 担心效果：我们可以约定 [X 目标]，[X 时间] 后复盘
  · 担心时间：可以调整到 [X 频率]
  · 担心费用：[X 课时包] 拆成 [X] 阶段付费也可以
  · 担心孩子状态：可以先续 [X 课时] 短期试一下

我们可以先 [X 动作]，看 [X 时间] 后孩子的反应。
最重要的是 [化名] 的状态和进步。
您看哪种方式最适合您？"
```

### 8.3 续费困难的处理

```text
处理原则：
  · 不强行挽留
  · 体面结束
  · 留下好印象（学员可能回流）

话术：
  "X 妈/爸，谢谢您这段时间对 [化名] 的支持。
  如果您觉得现在需要调整，我们尊重您的决定。
  [化名] 这段时间的进步是真实的：
  [X 进步 1] [X 进步 2] [X 进步 3]
  
  之后 [化名] 如果想回来继续学习，
  随时联系我。
  也欢迎把 [化名] 介绍给身边有需要的朋友。"
```

## 九、自检清单

每次输出阶段报告前，AI 必须自检：

- [ ] 报告是否基于真实数据（不编造）
- [ ] 三段式结构是否完整（事实+进步+计划）
- [ ] 进步展示是否有具体数据
- [ ] 计划是否给到具体动作
- [ ] 续费建议是否给到多个选项
- [ ] 续费沟通话术是否基于真实进步
- [ ] 是否避开了承诺提分
- [ ] 是否使用化名
- [ ] 是否给老师确认空间（不自动发送）
- [ ] 家长犹豫/困难的应对是否准备

---

> **使用提示**：建议老师在课时包消耗到 50% 时自动生成中期报告预览，到 70% 时生成续费报告；不要等到课时耗尽才匆忙准备。
