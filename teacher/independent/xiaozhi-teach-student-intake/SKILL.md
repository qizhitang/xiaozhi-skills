# 试听与学员建档 SKILL

> **一句话定位：** 试听不是销售环节，而是诊断环节——让老师真正看见这个学生，让家长真正理解这个老师。

---

## ⚠️ 技术实现边界声明

> **关于"学员信息收集"边界：** 本 SKILL 严格遵循**最小化收集原则**：
> ✅ 收集：化名、年级、学科、课时需求、家长联系方式（仅 1 个）
> ❌ 不收集：身份证号、家庭住址、家庭收入、家长职业
> 试听期间所有信息使用化名（化名 A/B/C 等），不暴露真实姓名。
>
> **关于"试讲课"边界：** 试讲课的核心是"诊断"而非"展示老师多厉害"；不夸大教学效果，不做"保证提分"承诺。
>
> **关于"自动跟进"边界：** 本 SKILL **不**自动发送消息给家长；所有跟进动作需要老师确认；AI 仅生成"跟进话术建议"和"跟进时间提醒"。

---

## 一、核心使命

独立教师试听环节常见的三个误区：

```text
误区① 销售化试听：把试听课上成"表演课"，
        老师表演精彩但学生学不到东西，
        家长试听后觉得"挺好"但孩子留下后不爱来。

误区② 一刀切建档：把学员档案当成简单的"姓名+年级"，
        没有需求画像、没有学情诊断，
        后续排课和教学全凭感觉。

误区③ 试听后无跟踪：试听完就结束，
        没有跟进节点、没有转化判断、
        家长/学生很快遗忘。
```

本 SKILL 要解决的是：
- **让试听成为双向诊断**：老师诊断学生 + 家长诊断老师
- **让学员档案是"教学起点"**：含需求画像 + 学情基线 + 目标设定
- **让跟进有节奏**：转化期/续费期/流失期 三个阶段不同话术

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4.2 不收字段

```text
❌ 身份证号
❌ 家庭住址（精确到门牌号）
❌ 家庭收入
❌ 家长职业（精确到岗位）
❌ 学生成绩单原件
❌ 学生户口本信息
❌ 家庭成员关系详情
```

### 4.3 化名规则

```text
试听期：用 A/B/C/D ... 顺序化名
正式学员期：使用"X 同学"格式
  · 例：化名 A → 转正后为"小米同学"或"小张同学"
  · 化名不暴露真实姓氏
  · 同一学员在所有文档中化名一致
```

## 五、需求访谈（5W 框架）

### 5.1 5W 提问清单

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 需求访谈 · 5W
时长：15-20 分钟
对象：家长主导 + 学生补充
━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ Why 为什么来
  · "孩子/您为什么想补习 X 学科？"
  · 期待解决的具体问题？
  · 之前有过什么尝试？

■ What 学什么
  · "希望重点突破哪些内容？"
  · 校内同步 vs 课外拔高 vs 应试准备？
  · 是否有具体目标（如期末考试/竞赛/升学）？

■ When 多久
  · "期望多长时间看到效果？"
  · 短期目标 vs 长期目标？
  · 是否有时间节点（如 3 个月后月考）？

■ Who 谁监督
  · "课后谁负责督促孩子？"
  · 家长能提供什么样的支持？
  · 学生自己是否有主动性？

■ Where 学习环境
  · "学生平时的学习环境？"
  · 有没有专门的书桌/独立房间？
  · 课后复习时间通常几点到几点？
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 5.2 学员本人访谈（针对高年级学生）

```text
针对初高中学生（>12 岁）：
  · "你自己觉得 X 学科哪里学得最吃力？"
  · "你希望老师上课是什么风格？"
  · "你喜欢什么样的老师？"
  · "你现在的学习时间是怎么安排的？"
  · "你有什么兴趣爱好？"

⚠️ 注意：
  · 家长在场的场合，学生可能不敢说真话
  · 必要时单独与学生沟通 2-3 分钟
  · 不替学生做"懂事"的判断
```

### 5.3 需求画像卡

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 需求画像 · [化名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 学习目标
  短期（1 个月内）：____________________________________
  中期（3 个月内）：____________________________________
  长期（学期内）：____________________________________

■ 讲解与节奏偏好（记录为可变偏好，非固定"学习风格"标签）
  讲解方式：先举例再讲道理 / 先讲框架再填细节 · 需要多举例 / 点到即可
  互动节奏：快问快答 / 慢一点深入讲
  难度节奏：接受难度跳跃 / 需要循序渐进
  （注：VAK"视觉型/听觉型/动手型学习风格"缺乏科学证据，不作为分型标签；此处只记具体可观察的偏好）

■ 家长期待值
  提分：[N] 分
  习惯培养：[具体行为]
  心理建设：[具体期待]
  评估家长期待合理性：合理 / 偏高 / 偏低

■ 学员学习动机
  内在兴趣：____ / 10
  外部压力：____ / 10
  紧急程度：____ / 10

■ 障碍预判
  时间：□充足 □紧张
  注意力：□专注 □一般 □易分心
  家庭支持：□充分 □一般 □不足
```

## 六、学情诊断（前测评）

### 6.1 诊断三步走

```text
第 1 步：访谈获取信息
  · 当前成绩（粗略即可，不强求精确分数）
  · 之前补习经历
  · 校内老师反馈

第 2 步：5-10 分钟前测评
  · 选 2-3 道题覆盖核心知识点
  · 看答题过程（不是只看答案）
  · 看：思路、计算、书写规范

第 3 步：访谈式追问
  · "这道题你是怎么想到这个方法的？"
  · "X 知识点你还记得吗？"
  · "如果把 Y 改成 Z 呢？"
```

### 6.2 学科前测评（示例：数学）

```text
基础版（5 分钟）：
  1. 计算题（看计算功底）
  2. 一道已学章节的简单题（看知识留存）
  3. 一道略超纲的题（看思维弹性）

标准版（10 分钟）：
  1. 计算题 2 道
  2. 概念辨析 1 道
  3. 综合应用 1 道
  4. 开放题 1 道（看思路深度）
```

## 七、试讲课设计

### 7.1 试听三原则

```text
原则 ① 诊断式而非表演式
  · 试讲课的核心是"看见学生"而非"展示老师"
  · 不必追求课程完美，重点观察学生反应

原则 ② 双向契合度判断
  · 老师看：学生的学情是否在我能力范围
  · 家长看：老师的风格是否匹配孩子
  · 学生看：这位老师我是否喜欢

原则 ③ 真实而非包装
  · 试讲课的教学风格应与正式课一致
  · 不夸大、不掩饰
  · 真实让学生有预期，正式课不会有落差
```

### 7.2 试讲课 5 段结构

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 试讲课设计 · [化名] · [日期] · 60-90 分钟
━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 段 1：暖场（5-10 分钟）
  · 自我介绍（老师 + 学生）
  · 聊兴趣、聊学校
  · 缓和紧张，建立信任

■ 段 2：诊断（10-15 分钟）
  · 1-2 道前置测评题
  · 观察：思维路径、计算、表达
  · 决定试讲课的起点

■ 段 3：核心教学（25-35 分钟）
  · 选 1 个核心知识点
  · 完整讲授：讲解 → 示范 → 学生尝试
  · 观察：注意力、提问、反应

■ 段 4：互动评估（10-15 分钟）
  · 1-2 道变式题
  · 观察：迁移能力
  · 看学生是否真懂

■ 段 5：收尾与跟进（5-10 分钟）
  · 总结今天学了什么
  · 给孩子具体表扬
  · 与家长 5 分钟简短沟通
  · 约定下次课时间
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 7.3 试讲课选材原则

```text
✅ 选材原则
  · 选"中等难度+稍有兴趣"的内容
  · 选能展示学生真实状态的题
  · 选 1-2 个知识点即可，不贪多

❌ 避免
  · 选太简单（学生无聊）
  · 选太难（学生受挫）
  · 选太泛（学生不知学什么）
```

## 九、转化/续费/流失三阶段跟进

### 9.1 转化期（试听后 1-7 天）

```text
目标：从"试听"到"正式学员"
节奏：
  试听后当天：发简短反馈（家长 + 学生各 1 份）
  试听后 2-3 天：问候 + 答疑
  试听后 5-7 天：转化话术

话术示例（试听后 2-3 天）：
  "[X 妈，您好！上次试听后，[化名] 学得怎么样？
  如果有疑问随时问我。
  如果觉得我们的教学风格合适，可以先约 [N] 课时小包试试。]"

⚠️ 避免：
  · 频繁催报
  · 强行推销课时包
  · 贬低其他机构/老师
```

### 9.2 续费期（课时包消耗到 70%-80%）

```text
目标：让学员在"课时耗尽"前续费
节奏：
  课时消耗 50%：发学习进度报告
  课时消耗 70%：主动沟通续费
  课时消耗 80%：续费节点预警

话术示例（70% 续费节点）：
  "[X 妈，[化名] 课时还剩 [N] 课时。
  这段时间孩子在 [X] 方面进步明显（具体说明）。
  接下来 [下一步学习计划]。
  您看是按 [建议课时包] 续费，还是其他安排？]"
```

### 9.3 流失期（连续 2-3 次缺课/退费倾向）

```text
目标：诊断流失原因，挽留或体面结束
节奏：
  缺课 1 次：主动问候
  缺课 2 次：了解原因
  缺课 3 次：判断是否流失

话术示例（连续 2 次缺课后）：
  "[X 妈，注意到 [化名] 这两次没来上课。
  孩子是身体不适还是其他原因？
  如果时间安排有变化，我们可以一起调整。
  如果您觉得现在的安排需要调整，也欢迎告诉我。]"

⚠️ 注意：
  · 不强行挽留
  · 体面结束，留好印象（学员可能回流）
  · 不诋毁、不抱怨
```

### 11.1 协作流图

```text
            ┌────────────────────────┐
            │ 学员/家长提供信息        │
            └───────────┬────────────┘
                        ↓
            ┌────────────────────────┐

  solo-dashboard   student-analyzer  lesson-planner
  （学员档/排课）  （学情 baseline） （首课教案）
```

### 11.2 接口

> 所有读写均落在共享工作空间 `solo-teacher-workspace.schema.json`；下列为真实存储字段，派生/生成项已单独标注。

```text
读：（本 SKILL 位于流程起点，无前置 SKILL 数据依赖；建档/写入前须校验目标学员的
     workspace.studentCards[].consent.profileEnabled / crossSkillSharing 为 true）

写：
  学员档案     → workspace.studentCards[]
                （studentId / alias / gradeLevel / subjects[] /
                  primaryWeaknesses[] / learningPreferences[] /
                  guardianCommunicationPreference / consent{...}）
  学习目标     → workspace.studentCards[].goals[]
  上课时间偏好 → workspace.lessonSchedule[].startTime / .day_of_week
                （首课排期时逐条写入 lessonEvent）
  课时包       → workspace.coursePackageLedger[]
                （packageId / totalUnits / usedUnits / remainingUnits /
                  renewalAttention）
  学情基线证据 → workspace.progressEvidence[]
                （evidenceType / description / confidenceLevel，源自试听诊断）

  风险标记     →（派生视图，非存储字段：由 solo-dashboard 依据
                  workspace.homeworkFollowups[].status（作业风险）、
                  workspace.lessonLogs[].masteryStatus（测评退步）、
                  workspace.coursePackageLedger[].remainingUnits /
                  .renewalAttention（课时耗尽预警）实时计算，本 SKILL 仅在
                  试听记录中给出口头判断，不写入独立字段）

  → solo-dashboard 读取 studentCards[] / coursePackageLedger[] 显示学员档与排课
  → student-analyzer 读取 primaryWeaknesses[] / progressEvidence[] 作为学情基线
  → lesson-planner 读取 studentCards[].goals[] / lessonSchedule[] 生成首课教案
```

## 十二、字段级高敏信息防护

### 12.1 最小化原则

```text
✅ 收集：
  化名、年级、学科、课时、家长联系方式（1 个）
  获客渠道、家长称呼、紧急联系人

❌ 不收集：
  身份证号、家庭住址、家庭收入、家长职业
  学员成绩单原件、家庭成员关系
```

### 12.2 化名一致性

```text
试听期：化名 A、B、C...
正式期：化名"小米""小张"
所有文档化名一致
公开报告/案例绝不暴露真实姓名
```

### 12.3 写回 solo-dashboard 的脱敏

```text
✅ 写：化名 + 学情 baseline
❌ 不写：真实姓名 + 联系方式
✅ 写：课时包到期日
❌ 不写：家庭财务信息
```

## 十三、行为准则

| ✅ 应该做 | ❌ 不能做 |
|---------|---------|
| 最小化收集学员信息 | 收集身份证号/家庭住址 |
| 试听是诊断而非表演 | 把试课上成"表演课" |
| 双向契合度判断 | 单方面销售课时包 |
| 跟进有节奏 | 频繁催报或强行推销 |
| 流失期体面结束 | 强行挽留/诋毁其他 |
| 化名一致使用 | 公开档案/案例暴露真实姓名 |
| 写回数据用化名 | 写回真实姓名+联系方式 |

## 十四、与其他 SKILL 的协同清单

```text
试听与学员建档
    ──→ xiaozhi-teach-solo-dashboard（学员档/排课）
    ──→ xiaozhi-teach-student-analyzer（学情基线）
    ──→ xiaozhi-teach-lesson-planner（首课教案）
```

**禁止行为**：

- 禁止"保证提分"承诺
- 禁止在试讲课中夸大教学效果
- 禁止自动发送跟进消息（需老师确认）
- 禁止公开档案/案例暴露真实姓名

## 隐私与数据控制入口

> 本 SKILL 读写的学员数据存于共享工作空间（`solo-teacher-workspace.schema.json`），涉及未成年人信息，须提供可执行的控制入口。老师本人、或应学员/家长要求，可随时说：

- **查看**："查看 [学员化名] 的工作空间记录 / 课表 / 作业 / 报告"
- **更正**："更正 [学员化名] 的 [某字段]"（覆盖旧值，避免新旧冲突并存）
- **删除**："删除 [学员化名] 的某条记录 / 全部数据"（流失学员应按约定周期删除）
- **暂停记录**："这次不要记录 / 暂停记录 [学员化名]"
- **取消跨 SKILL 共享**："不要把 [学员化名] 的数据共享给其他 SKILL"

**校验要求**：跨 SKILL 共享或建档前，须确认 `consent.crossSkillSharing` / `consent.profileEnabled` 为 true；涉及未成年人敏感信息（真实姓名、出生年月、联系方式等）须经监护人单独同意，默认不收集、不写入（详见 `SECURITY_BASELINE.md`）。
