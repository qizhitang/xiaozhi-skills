# xiaozhi-teach-student-intake (Lite)

> 精简版 SKILL，完整定义与示例请参阅 `SKILL.md`。

## 触发条件

| 触发词 | 动作 |
|---|---|
| 新学员要试听 / 安排试听 | 触发 5 段试听流程 |
| 学员建档 / 新学员录档 | 触发正式档案建立 |
| 家长/学生有什么需求 | 触发 5W 需求访谈 |
| 试听课怎么上 / 试听讲什么 | 触发试讲课设计 |
| 试听完怎么记录 | 触发 5 维度观察记录 |
| 试听后怎么跟进 / 怎么转化 | 触发转化期/续费期/流失期三阶段 |
| 要不要续费 / 学员能继续上吗 | 触发续费判断 |
| 学员不来上课 / 学员退费 | 触发流失期体面结束 |

## 核心流程骨架

```text
收集基本信息（最小化）→ 5W 需求访谈
→ 学情诊断（前测评） → 试讲课 5 段设计
→ 5 维度试听记录 → 转化/续费/流失三阶段跟进
→ 正式学员档案 → 写回 solo-dashboard
```

## 输出要点

- 基本信息卡（化名/年级/学科/课时/家长 1 个联系方式）
- 5W 需求访谈清单
- 学情诊断卡（5-10 分钟前测评）
- 试讲课 5 段设计（暖场/诊断/核心教学/互动评估/收尾）
- 5 维度试听记录
- 转化/续费/流失三阶段话术
- 正式学员档案

## 禁止行为

| 禁止项 | 说明 |
|---|---|
| 收集身份证号/家庭住址/家庭收入 | 严格最小化原则 |
| 收集成绩单原件/家庭成员详情 | 仅需化名/年级/学科/课时 |
| "保证提分"承诺 | 试听只做诊断，不做承诺 |
| 试课上成表演课 | 核心是看见学生而非展示老师 |
| 自动发送跟进消息 | 跟进话术需老师确认后发出 |
| 强行推销/频繁催报 | 跟进有节奏，不强买强卖 |
| 公开档案暴露真实姓名 | 化名一致使用 |
