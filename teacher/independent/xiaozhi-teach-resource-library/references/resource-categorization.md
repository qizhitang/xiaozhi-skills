## 一、资源五大类

### 1.1 讲义库

```text
■ 用途
  · 课前发给学员的预习材料
  · 课中使用的讲义
  · 课后复习材料

■ 入库要求
  · 学科 + 章节
  · 难度（基础/中等/提升/挑战）
  · copyrightStatus
  · 适用学员类型
  · 关联学员（化名）

■ 示例
  ━━━━━━━━━━━━━━━━━━━━━━━━━━
  讲义 ID：H001
  标题：一次函数图象与性质
  学科：数学 · 一次函数
  难度：中等
  copyrightStatus：自有
  适用学员：初一/初二基础-中等
  关联学员：化名 A、化名 C
  关联资源：教案 L001、题库 Q001-Q005
```

## 二、标签化体系

```text
┌──────────┬──────────────┬──────────────┬──────────────┐
│ 维度      │ 标签值示例                    │ 用途          │
├──────────┼──────────────────────────────┼──────────────┤
│ 学科      │ 数学/语文/英语/物理/...       │ 主分类        │
│ 章节      │ 函数/几何/语法/...            │ 精确检索      │
│ 知识点    │ 一次函数图象/定语从句/...      │ 精确定位      │
│ 难度      │ 基础/中等/提升/挑战          │ 学员匹配      │
│ 学员类型  │ 基础学员/拔尖学员/...        │ 复用建议      │
│ 学段      │ 小学/初中/高中/...           │ 适用筛选      │
│ 标签      │ 优秀/经典/必看/...            │ 主观标记      │
└──────────┴──────────────────────────────┴──────────────┘
```

### 2.1 标签使用规则

```text
■ 主标签（必填）
  · 学科 + 章节 + 知识点
  · 难度

■ 辅标签（推荐）
  · 学员类型
  · 标签（优秀/经典等）

■ 自定义标签（可选）
  · 老师个人标签
  · 跨学科联结
```

## 三、检索与匹配

### 3.1 关键词检索

```text
输入：关键词（如"一次函数复习"）
匹配：所有包含"一次函数"+"复习"的资源
排序：按相关度 / 使用频率 / 学员匹配度
```

### 3.2 学员画像匹配

```text
输入：学员画像（化名 + 学情摘要 + 需求）
匹配：基于画像推荐最相关的资源
输出：
  · 讲义（适合 X 学段 X 难度）
  · 题库（针对 X 知识点）
  · 讲评话术（针对 X 错因）
  · 错因案例（参考类似学员）
```

### 3.3 错因案例检索

```text
输入：错因（如"概念模糊"）
匹配：所有相关错因案例
输出：
  · 案例描述
  · 应对动作
  · 效果记录
```

## 四、复用与改编

### 4.1 复用四步走

```text
第 1 步：检索资源
  · 基于当前需求（学员画像/学情）检索
  · 找到 1-3 个候选资源

第 2 步：评估匹配度
  · 难度匹配？
  · 知识点匹配？
  · 学员类型匹配？

第 3 步：调整适配
  · 修改参数（数字/情境）
  · 调整难度
  · 添加学员特定元素

第 4 步：使用并记录
  · 标记"复用自 [资源 ID]"
  · 记录使用效果
  · 反馈到资源库
```

### 4.2 改编四原则

```text
■ 改数（参数）
  原题 y=2x+1 → 改编 y=3x-2
  适合：同一知识点不同参数

■ 改问（设问角度）
  原题"求 X" → 改编"判断 X 是否正确"
  适合：换角度测同一概念

■ 改情境（背景）
  原题"出租车" → 改编"网约车"
  适合：让题目更贴近学生生活

■ 改综合度
  单知识点 → 多知识点综合
  适合：测评综合应用能力
```

### 4.3 复用记录表

```text
┌──────────┬────────────┬────────────┬────────────┐
│ 资源 ID  │ 使用次数   │ 平均效果   │ 最近使用    │
├──────────┼────────────┼────────────┼────────────┤
│ H001     │ 5 次       │ 班级 78%   │ [日期]     │
│ Q001     │ 12 次      │ 正确率 72% │ [日期]     │
│ P001     │ 8 次       │ 有效 7/8   │ [日期]     │
└──────────┴────────────┴────────────┴────────────┘
```

## 五、版权管理

### 5.1 资源版权四档

```text
✅ 自有（copyrightStatus: own）
  老师原创或组内原创
  完整存储

✅ 改编（copyrightStatus: adapted）
  基于某资源改编
  必须标注"原出处 + 改编点"

✅ 公开可引用（copyrightStatus: public）
  教材例题、CC 协议资源
  完整存储，标注来源

⚠️ 仅存索引（copyrightStatus: index-only）
  教辅原题只记题号和出处
  不复制题干

❌ 禁止
  未授权复制教辅原题
  盗版扫描件
  未授权转载的网络资源
```

### 5.2 版权标注模板

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━
📄 资源版权信息
━━━━━━━━━━━━━━━━━━━━━━━━━━━
资源 ID：[   ]
copyrightStatus：[own/adapted/public/index-only]

【own】
  作者：[老师/组内]
  创建时间：[日期]

【adapted】
  原出处：[X 教辅/网站/资料]
  改编点：
    · [X 处]
    · [X 处]
  改编时间：[日期]

【public】
  来源：[X 教材/CC 协议资源]
  引用许可：[CC BY 4.0 等]

【index-only】
  题号：[X 教辅 第 X 页 题 X]
  出处：[X 教辅 出版社/作者]
  不复制题干
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 六、案例脱敏规则

### 6.1 必脱敏字段

```text
■ 必脱敏
  · 真实姓名 → 化名
  · 学校名称 → "某公立/民办学校"
  · 年级 → 模糊（"初中学员"）
  · 家长身份 → 移除
  · 家庭住址 → 移除
  · 联系方式 → 移除

■ 可保留（用于案例分析）
  · 学科
  · 知识点
  · 错因
  · 应对动作
  · 效果数据
```

### 6.2 脱敏检查清单

```text
□ 学员姓名已替换为化名
□ 学校名称已模糊化
□ 年级已模糊化（"初中学员"等）
□ 家长身份信息已移除
□ 家庭住址已移除
□ 联系方式已移除
□ 案例描述不含可识别细节
□ 案例分析基于错因/方法/效果而非具体人
```

## 七、持续更新机制

### 7.1 入库时机

```text
■ 必入库时机
  · 学员试讲课讲义通过后
  · 优秀作业讲评案例
  · 重要错因突破案例
  · 阶段性测评优秀试卷
  · 复习规划方案

■ 选入库时机
  · 课堂过渡话术
  · 家长沟通案例
  · 学员进步转折点
```

### 7.2 资源生命周期

```text
新入库 → 试用 → 优化 → 成熟 → 经典
  ↓
  归档（如过时/淘汰）
```

## 八、自检清单

每次入库/检索/复用资源前，AI 必须自检：

- [ ] 资源入库时是否打了主标签（学科/章节/难度）
- [ ] 学员案例是否完全脱敏（化名+模糊年级+去除可识别）
- [ ] 版权是否清晰标注（copyrightStatus）
- [ ] 检索是否基于画像/关键词/错因（非反查学员）
- [ ] 复用时是否做了调整适配
- [ ] 使用后是否记录了效果
- [ ] 资源是否标记了使用频次
- [ ] 是否避开了未授权教辅原题
- [ ] 是否给老师保留选择空间（不自动推送）

---

> **使用提示**：建议老师每次重要教学活动（试讲/复习/错题突破）结束后，立即把"讲义+题库+讲评话术+错因案例"打包入库；不要等"有空再整理"——等你回头整理，记忆已经模糊了。
