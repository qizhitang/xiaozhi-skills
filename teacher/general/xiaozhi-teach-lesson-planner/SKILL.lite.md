# xiaozhi-teach-lesson-planner (Lite)

> 精简版 SKILL，完整定义与示例请参阅 `SKILL.md`。

## 触发条件

| 触发词 | 动作 |
|---|---|
| 帮我设计一节课 / 写一份教案 | 进入 UbD 三阶段流程 |
| 这节课怎么上 / 教学设计 | 触发三维目标 + 六步矩阵 |
| 分层教案 / 基础班和提升班怎么上 | 触发 A/B/C 分层输出（需学情摘要） |
| 生成课堂提问链 / 学生不发言 | 触发 Bloom 六层提问链 |
| 这节课重做成 UbD 版本 | 重写现有教案为 UbD 结构 |
| 试卷讲评课 / 复习课 | 切换到讲评课或复习课模板 |

## 核心流程骨架

```text
确认主题与课时 → UbD Stage1 预期结果 → Stage2 评估证据
→ Stage3 学习经验（WHERE TO）→ 三维目标 → 六步环节
→ Bloom 提问链（≥4 层）→ A/B/C 分层 → 输出完整教案
```

## 输出要点

- 三维目标（知识与技能 / 过程与方法 / 情感态度价值观）
- 六步环节时间矩阵（导入 5 / 新授 15 / 练习 12 / 巩固 5 / 总结 3 / 作业 5）
- Bloom 提问链（覆盖至少 4 个层次，1 题 1 候场追问）
- A/B/C 分层任务卡（无学情时输出"无学情"基础版）
- 板书设计与风险预案

## 禁止行为

| 禁止项 | 说明 |
|---|---|
| 编造学情 | 没有学情数据时输出基础版，不编造 A/B/C 划分 |
| 写空泛目标 | "理解概念""培养思维"等不可观测的目标无效 |
| 一刀切教案 | 同一份任务卡不直接给三层学生用 |
| 超量输出 | 一次最多输出 3 个课时教案，避免老师无法消化 |
| 强加时间 | 必须基于实际课时（45/60/90/30）分配，不自动换算 |
