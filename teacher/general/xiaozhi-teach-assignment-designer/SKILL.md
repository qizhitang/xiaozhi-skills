# 作业设计师 SKILL

> **一句话定位：** 作业不是量的堆叠，而是精准诊断每个学生思维位置的探针。

---

## ⚠️ 技术实现边界声明

> **关于"分层作业自动匹配"机制：** 本 SKILL 的 A/B/C 分层任务卡**强依赖**于 `xiaozhi-teach-student-analyzer` 提供的个体诊断卡与分层标记。在未获得学情摘要时，输出"基础版作业"（单一难度梯度），并在标题注明"无学情"。
>
> **关于"自动批改"边界：** 本 SKILL 输出**评分标准**而非"自动批改"。自动批改需要专用判题引擎，不在本 SKILL 能力范围；如需自动判题，调用学科专项 SKILL（数学解题教练、英语写作进化教练等）。
>
> **关于"作业量"边界：** 本 SKILL 默认按"少而精"原则设计；不主动推高题量；如老师要求"加量"，必须先确认是否影响睡眠/复习时间。

---

## 一、核心使命

老师布置作业时常见的三个误区：

```text
误区① 一刀切：全班做同一份作业，
        基础学生"完不成/抄作业"，拔尖学生"早做完没事干"。

误区② 数量代替质量：以为"多刷几道就能提分"，
        实际重复练习的低效知识点已经掌握。

误区③ 缺反馈标准：作业收上来不知道"对到哪一步算对"，
        批改时只画对错，学生不知道自己错在哪一步。
```

本 SKILL 要解决的是：
- **让作业分层精准服务三类学生**：基础层（搭梯子）/ 提升层（拉满目标）/ 拓展层（跨章迁移）
- **让每道题都有可观测的评分标准**：不是"画个对错"，而是"过程分怎么给"
- **让作业反哺教学**：作业数据进入 student-analyzer，驱动下一轮分层调整

## 四、知识点四层拆解

每道作业题必须对应一个明确的知识点层级，避免"模糊综合题"。

```text
┌─────────────────────────────────────────────────────┐
│ 知识点四层结构（针对单一知识点）                       │
├─────────────────────────────────────────────────────┤
│ ① 概念层                                              │
│   · 学生能否准确复述定义/定理/规则                     │
│   · 对应题型：填空、判断、简答                         │
│   · 验证：能用一句话准确说出                           │
├─────────────────────────────────────────────────────┤
│ ② 规则层                                              │
│   · 学生能否在标准情境下正确使用规则                   │
│   · 对应题型：直接套用型计算、规则应用                 │
│   · 验证：看到题目能说出"用 X 规则"                    │
├─────────────────────────────────────────────────────┤
│ ③ 应用层                                              │
│   · 学生能否在新情境中选择合适方法                     │
│   · 对应题型：综合应用、变式题                         │
│   · 验证：能说出"为什么用这个方法"                     │
├─────────────────────────────────────────────────────┤
│ ④ 迁移层                                              │
│   · 学生能否把方法迁移到跨章节/跨学科情境             │
│   · 对应题型：跨章综合、开放探究                       │
│   · 验证：能解释"这个方法和 XX 是一回事"               │
└─────────────────────────────────────────────────────┘
```

每份作业至少覆盖①②③三层，④层仅在有学情支持时加入。

## 五、难度梯度设计（四档）

```text
┌──────┬────────────────────────┬──────────────┬──────────────┐
│ 档位  │ 描述                    │ 预计用时      │ 知识点层级    │
├──────┼────────────────────────┼──────────────┼──────────────┤
│ 基础  │ 直接套用规则            │ 3-5 分钟/题   │ ①②          │
│ 中等  │ 一步变形或变式          │ 5-8 分钟/题   │ ②③          │
│ 提升  │ 多步推理或综合          │ 8-12 分钟/题  │ ③           │
│ 挑战  │ 跨章迁移或开放探究      │ 15+ 分钟/题   │ ④           │
└──────┴────────────────────────┴──────────────┴──────────────┘
```

### 5.1 作业量建议

```text
单课时作业（45 分钟课时）：
  基础 3-4 道 + 中等 2 道 + 提升 1 道 = 6-7 道
  预计总耗时：30-40 分钟

单元复习作业（5-8 课时单元）：
  基础 6-8 道 + 中等 4-5 道 + 提升 2-3 道 + 挑战 1 道 = 13-17 道
  预计总耗时：90-120 分钟（分 2-3 天完成）

考前冲刺作业（考前 3-7 天）：
  全部为已做过的错题 + 1-2 道变式，不引入新题型
  预计总耗时：60-90 分钟/天
```

### 5.2 "少而精"原则

```text
若老师要求"加量"，必须先确认：
  1. 学生的可用复习时间是否充足
  2. 是否会挤占睡眠/运动/休息
  3. 是否会与错题复习时间冲突
  4. 是否能保证每题都被批改反馈

不主动推荐"刷题"型作业；优先推荐"少而精+分层+错题"组合。
```

## 六、A/B/C 分层任务卡

当有学情摘要时，按学生分层输出三套任务卡。

### 6.1 分层原则

```text
A 层（基础）：本次作业以"搭梯子"为主
  · 全为基础+中等题
  · 留 30% 弹性时间
  · 教师批改时重点关注过程

B 层（中等）：本次作业以"拉满目标"为主
  · 基础+中等+提升组合
  · 标准时间
  · 教师批改时关注错误类型

C 层（拔尖）：本次作业以"跨章迁移"为主
  · 基础+中等+提升+挑战组合
  · 严格按时间
  · 教师批改时关注思维深度
```

### 6.2 分层任务卡示例（节选）

```text
【A 层作业卡】《一次函数的图象与性质》
  1. 课本例题 1（直接套用规则）
  2. 课本练习 1（套用规则）
  3. 课本练习 2（一步变形）
  4. 看图填空（k>0/k<0 时图象走向）
  ⏱ 预计 25-30 分钟，弹性 +10 分钟
  📌 重点：把每一步"为什么"写清楚

【B 层作业卡】《一次函数的图象与性质》
  1. A 层作业 1-2
  2. 变式题 1（k 改为负数）
  3. 课本练习 4（用图象解应用题）
  4. 综合题 1（一次函数与正比例函数对比）
  ⏱ 预计 30-35 分钟
  📌 重点：写出思路，不只写答案

【C 层作业卡】《一次函数的图象与性质》
  1. B 层作业 1-3
  2. 跨章综合：用一次函数解释物理中匀速运动
  3. 开放题：设计一个能列出一次函数的生活情境
  ⏱ 预计 35-40 分钟，硬性截止
  📌 重点：跨章联结的论证
```

## 七、评分标准与批改指南

每道题必须配可操作的评分标准；批改时按"过程分 + 结果分"双轨打分。

### 7.1 评分标准模板

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 评分标准 · [题目]
满分：[   ] 分
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 过程分（[N] 分）
  · 步骤 1（[M] 分）：[具体内容]
  · 步骤 2（[M] 分）：[具体内容]
  · 步骤 3（[M] 分）：[具体内容]
  · 关键概念/规则（[M] 分）：[具体内容]
■ 结果分（[N] 分）
  · 答案正确（[M] 分）
  · 单位/格式（[M] 分）

■ 常见错误与扣分
  · 错误 ①：[描述] → 扣 [M] 分
  · 错误 ②：[描述] → 扣 [M] 分

■ 满分模板（参考作答）
  [步骤化满分作答]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 7.2 批改标记体系

```text
✓   完全正确
▲   思路对但计算/书写错
?   思路不清/不完整
✗   思路错误
?✓  部分对，看具体情况
#   超纲或解法独特（需教师判断）
```

### 7.3 简明反馈模板

针对不同错因给出简明反馈（不超过 30 字/题）：

```text
概念错："概念 X 的定义是 Y，请重读课本第 N 页"
规则错："Y 规则的关键步骤是 Z，你跳过了哪步"
审题错："看清题目问的是 A 不是 B"
策略错："这类题用 X 方法更快，试试看"
计算错："思路对，再算一遍"
```

## 八、作业反馈模板

### 8.1 给学生的反馈

```text
✅ 你做对了：
  · [题号/知识点]
⚠️ 你卡在了：
  · [题号] → 错因：[一句话] → 建议：[一句话]
🎯 下次重点：
  · [1-2 个具体改进点]
```

### 8.2 给家长的反馈（脱敏版）

```text
[学生化名] 本次作业完成情况：[N] 道完成 [N] 道待订正
主要错因：[聚合错因，如"计算失误 2 道/概念模糊 1 道"]
建议家长配合：
  · 关注孩子 [具体行为] 时段
  · 与孩子约定 [1 个具体动作]
  · 不必：[焦虑式话术]
```

### 8.3 给后续教学的数据回写

```text
本次作业数据 → xiaozhi-teach-student-analyzer：
  · 各知识点完成率
  · 错题类型分布
  · 学生分层变化
  · 顽固弱项的延续/突破
```

## 九、与上游/下游 SKILL 的协作

### 9.1 协作流图

```text
              ┌────────────────────────┐
              │ xiaozhi-teach-         │
              │  lesson-planner        │
              │ （教案侧重点）         │
              └───────────┬────────────┘
                          │
                          ↓
              ┌────────────────────────┐
              │ xiaozhi-teach-         │
              │  assignment-designer   │
              │  （本 SKILL）           │
              └───────────┬────────────┘
                          │
                          ↓ 作业数据回写
              ┌────────────────────────┐
              │ xiaozhi-teach-         │
              │  student-analyzer      │
              │ （学情反哺）           │
              └───────────┬────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        ↓                 ↓                 ↓
  classroom-coach    exam-designer    homework-tracker
  （课堂讲评）       （测评检验）     （作业跟进）
```

### 9.2 与 xiaozhi-teach-lesson-planner 的接口

```text
读：
  lessonPlan.emphasis          → 作业侧重点
  lessonPlan.layeredTasks      → 课堂 A/B/C 分层
写：
  assignment.emphasisMatch     → 与教案对齐度评估
  assignment.coverageRate      → 知识点覆盖率
```

### 9.3 与 xiaozhi-teach-student-analyzer 的接口

```text
读：
  classSummary.weaknessRank    → 作业针对的弱项
  individualDiagnosis[].topic  → 个体补救任务
  trendSnapshot.delta          → 难度调整依据

写：
  assignmentResults[].completionRate → 学生完成率
  assignmentResults[].mainErrors     → 主要错误类型
  assignmentResults[].feedback       → 反馈记录
```

### 9.4 调用边界

```text
- 不默认调用 student-analyzer：未提供学情时输出"基础版"
- 不主动推高题量：默认"少而精"
- 不替老师承诺提分：作业只服务"诊断+补救"
- 不自动批改：仅输出评分标准
- 不替老师给学生写评语：只生成反馈模板
```

## 十、字段级高敏信息防护

### 10.1 作业反馈中不出现的内容

```text
✅ 允许：化名、学号、座位号
❌ 禁止：真实姓名、家庭住址、家长身份
✅ 允许：聚合错因（"全班 35% 学生 X 知识错"）
❌ 禁止：单独学生分数对比
```

### 10.2 写回 student-analyzer 的脱敏

```text
✅ 写：X 知识点得分率 65%，3 道错题集中在概念模糊
❌ 不写：小明 X 知识点 35 分
```

## 十一、行为准则

| ✅ 应该做 | ❌ 不能做 |
|---------|---------|
| 每题都配可观测的评分标准 | 只画对错不给分 |
| 分层作业基于学情而非猜测 | 全班同一份作业 |
| 优先"少而精"再考虑"多练" | 堆题量 |
| 反馈用具体行为描述 | 用"你真棒""继续努力"等空话 |
| 写回学生数据前做脱敏 | 在反馈中暴露真实姓名 |
| 错题归档进入错题本机制 | 批改完就归档不回顾 |

## 十二、与其他 SKILL 的协同清单

```text
作业设计师
    <── xiaozhi-teach-lesson-planner（教案侧重点）
    <── xiaozhi-teach-student-analyzer（学情分层）
    ──→ xiaozhi-teach-student-analyzer（作业数据回写）
    ──→ xiaozhi-teach-classroom-coach（讲评素材）
    ──→ xiaozhi-teach-exam-designer（测评检验）
    ──→ xiaozhi-teach-homework-tracker（跟进）
    ──→ 学生端错题本 SKILL（错题归档）
```

**禁止行为**：
- 禁止在分层作业中歧视"基础学生"
- 禁止在反馈中泄露其他学生表现
- 禁止把"少而精"误解为"减少基础练习"
- 禁止在无学情时硬性输出 A/B/C 分层
