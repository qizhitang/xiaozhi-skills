# 双向细目表模板

> 配合 `xiaozhi-teach-math-exam-designer` 使用。让测评从"凭感觉"到"系统化"。

## 1. 双向细目表定义

```text
横轴：知识点
纵轴：认知层级（Bloom 分类）
交叉点：题目
```

## 2. Bloom 认知层级

```text
┌──────────┬────────────────────────┬──────────────┐
│ 层级      │ 描述                    │ 题型          │
├──────────┼────────────────────────┼──────────────┤
│ 记忆      │ 记住事实/概念            │ 选择/填空     │
│ 理解      │ 解释/归纳               │ 简答/判断     │
│ 应用      │ 用知识解决问题           │ 计算/解答     │
│ 分析      │ 拆解/比较               │ 综合/分析     │
│ 评价      │ 评判/论证               │ 论述/比较     │
│ 创造      │ 综合/创新               │ 探究/开放     │
└──────────┴────────────────────────┴──────────────┘
```

## 3. 双向细目表设计原则

```text
■ 知识点覆盖
  · 本单元所有重要知识点
  · 重点知识点题量多
  · 次要知识点题量少

■ 认知层级分布
  · 记忆 + 理解 ≥ 50%
  · 应用 + 分析 ≥ 35%
  · 评价 + 创造 ≤ 15%

■ 难度梯度
  · 基础题 60%
  · 中等题 30%
  · 提升题 10%
```

## 5. 双向细目表自检

```text
□ 知识点是否覆盖本单元所有重要点？
□ 重点知识点题量是否足够？
□ 认知层级分布是否合理？
□ 难度梯度是否合理（60/30/10）？
□ 总分是否 100？
□ 每道题分值是否合理？
```

## 6. 题目设计

### 6.1 题目选编

```text
┌──────────┬────────────────────────┬──────────────┐
│ 来源      │ 描述                    │ 适用          │
├──────────┼────────────────────────┼──────────────┤
│ 教材原题  │ 教材课后题              │ 基础训练      │
│ 改编题    │ 基于教材/经典题改编     │ 针对本班      │
│ 自有题    │ 老师原创                │ 班级特色      │
│ CC 协议  │ 公开可引用              │ 拓展          │
└──────────┴────────────────────────┴──────────────┘
```

### 6.2 题目设计原则

```text
■ 难度梯度合理
  · 基础→中等→提升
  · 学员有"上手感"

■ 知识点覆盖全
  · 双向细目表

■ 认知层级有梯度
  · 记忆→理解→应用
  · 不全是记忆

■ 题目情境真实
  · 学员有代入感

■ 避免
  · 偏题怪题
  · 套路题（答案唯一性差）
  · 信息超量
```

### 6.3 题目版权

```text
✅ 教材原题：标注来源
✅ 改编题：基于教材/经典题改编
✅ 自有题：老师原创
✅ CC 协议：公开可引用

❌ 禁止：未授权复制教辅题库
❌ 禁止：未授权转载网络题库
```

## 7. 双向细目表使用流程

```text
① 列出本单元所有重要知识点
  · 根据教学大纲
  · 根据本班教学实际

② 确定每个知识点的认知层级
  · 学员应达到什么层级

③ 设计题目
  · 按双向细目表设计
  · 标注每道题的知识点+认知层级

④ 检查分布
  · 知识点是否覆盖
  · 认知层级是否合理
  · 难度梯度是否合理

⑤ 调整
  · 不合理的地方调整
```
