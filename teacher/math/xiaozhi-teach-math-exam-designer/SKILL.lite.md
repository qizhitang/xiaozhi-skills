# 数学测评设计 · 精简版

> 详细版见 `SKILL.md`。

## 1. 一句话定位

测评不是给学员"打标签"，而是给教学"照镜子"——双向细目表是从"出题感觉"到"诊断精准"的桥梁。

## 2. 工作流

```text
测评目标 → 双向细目表 → 题目选编 → 难度梯度 → 测评实施 → 结果分析
```

## 3. 三类测评

```text
诊断性：找起点（学期初）
形成性：调进度（单元中）
总结性：评成果（学期末）
```

## 4. 双向细目表

```text
横轴：知识点
纵轴：认知层级（记忆/理解/应用/分析/评价/创造）
交叉点：题目
```

## 5. 难度梯度

```text
基础 60%（单一知识点/方法）
中等 30%（多个知识点/方法）
提升 10%（综合/创新/开放）
```

## 6. 题目来源

```text
教材原题 / 改编题 / 自有题 / CC 协议
（标注 copyrightStatus）
```

## 7. 测评频率

```text
诊断性：每学期 1-2 次
形成性：每单元 1 次
总结性：每学期 2-3 次
不频繁测评（学员压力）
```

## 8. 必触发场景

- 老师："这周考什么"
- 老师："试卷怎么出"
- 老师："双向细目表"
- 老师："诊断性测评"
- 老师："形成性测评"
- 老师："测评结果怎么用"
