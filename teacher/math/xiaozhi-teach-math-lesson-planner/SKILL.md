# 数学教案设计 SKILL

> **一句话定位：** 数学不是教学生"记住公式"，而是教学生"用公式思考"——概念建构是核心。

---

## ⚠️ 技术实现边界声明

> **关于"AI 备课"边界：** 本 SKILL 输出**教案框架**与**概念建构路径**；**不**代替老师备课，**不**输出"标准答案"。
>
> **关于"题目版权"边界：** 题目必须标注 copyrightStatus（教材原题/改编/自有/CC 协议）；不复制未授权教辅题库。
>
> **关于"AI 讲题"边界：** 本 SKILL **不**替代老师讲题；只提供**思路引导框架**和**变式训练设计**。

---

## 一、核心使命

数学教学常见的三个误区：

```text
误区① 概念=背诵：把概念当定义背诵，学员记不住、
        不会用。考完就忘。

误区② 讲题=念答案：老师在黑板上讲一遍，学员抄一遍，
        学员没有"自己思考"的过程。

误区③ 训练=刷题：只追求"量"，不讲方法、
        不讲变式、不讲关联。
```

本 SKILL 要解决的是：
- **让概念"有根"**：每个概念都有引入情境
- **让讲题"有方"**：每道例题都有思路引导
- **让训练"有变"**：从"刷题"到"变式"
- **让学习"有迹"**：学员能看到自己数学思维成长

## 四、概念建构

### 4.1 概念建构四步

```text
┌──────────┬────────────────────────┬──────────────┐
│ 步       │ 描述                    │ 关键问题      │
├──────────┼────────────────────────┼──────────────┤
│ ① 情境    │ 引入真实/模拟情境        │ 学员看到了什么│
│ ② 抽象    │ 提取共同特征            │ 什么是一样的 │
│ ③ 命名    │ 给概念命名              │ 叫什么        │
│ ④ 辨析    │ 区别于其他概念          │ 不是什么的   │
└──────────┴────────────────────────┴──────────────┘
```

### 4.2 情境引入 5 类

```text
① 生活情境
  · 学员生活中有真实体验
  · 例：分数（切披萨）、小数（钱）

② 历史情境
  · 数学史故事
  · 例：负数（古代记账）、圆周率（祖冲之）

③ 游戏情境
  · 让学员玩中学
  · 例：找规律（数字接龙）

④ 问题情境
  · 学员需要解决的具体问题
  · 例：行程问题（相遇追及）

⑤ 反例情境
  · 学员"试错"中理解
  · 例：平行（看上去平但不平行）
```

### 4.3 概念对比辨析

```text
■ 容易混淆的概念配对
  · 平方/立方
  · 锐角/钝角
  · 奇数/偶数
  · 质数/合数
  · 因数/倍数
  · 等腰/等边
  · 半径/直径
  · 周长/面积
  · 速度/时间/路程
  · 比率/比例
  · 概率/统计

■ 辨析三问
  · 这两个概念的共同点？
  · 这两个概念的差异点？
  · 怎么区分它们？
```

## 五、例题示范

### 5.1 例题三要素

```text
① 思路
  · 怎么想到的？
  · 关键是什么？

② 步骤
  · 第一步：X
  · 第二步：X
  · ...

③ 易错
  · 学员会错在哪里？
  · 怎么避免？
```

### 5.2 思路引导法

```text
■ "出声思维"
  · 老师边讲边"出声"
  · "我看到 X，所以想到 Y..."

■ "回溯法"
  · 从问题倒推
  · "要解 X，需要先知道 Y..."

■ "类比法"
  · 联系学员已知
  · "我们学过的 X 和这题有什么相同？"

■ "图示法"
  · 把抽象变形象
  · 线段图/示意图/坐标图

■ "假设法"
  · 先假设一个简单值
  · 看看会发生什么
```

## 六、变式训练

### 6.1 变式三类型

```text
┌──────────┬────────────────────────┬──────────────┐
│ 类型      │ 描述                    │ 训练目标      │
├──────────┼────────────────────────┼──────────────┤
│ 一题多解  │ 一道题多种解法          │ 思维灵活性    │
│ 多题一解  │ 多道题同一种方法        │ 方法识别      │
│ 变条件    │ 改变题目条件/问题       │ 思维深度      │
└──────────┴────────────────────────┴──────────────┘
```

### 6.2 一题多解设计

```text
■ 设计原则
  · 解法之间有差异
  · 解法有不同适用范围
  · 不为了多解而多解

■ 教学法
  · 老师先示范 2 种解法
  · 学员尝试第 3 种
  · 小组讨论"哪种最好"
```

### 6.3 多题一解设计

```text
■ 设计原则
  · 题目的"形"不同
  · 核心方法相同
  · 难度梯度合理

■ 教学法
  · 学员做 3-5 道题
  · 找出"共同点"
  · 总结方法
```

### 6.4 变条件设计

```text
■ 改条件
  · 把整数改成小数
  · 把一个条件去掉
  · 加一个条件

■ 改问题
  · 把"求 X"改成"判断 X 是否..."

■ 改情境
  · 把"鸡兔同笼"改成"鸡兔同笼 2.0"
```

```text
┌──────────┬────────────────────────┐
```

```text

```

## 十一、行为准则

| ✅ 应该做 | ❌ 不能做 |
|---------|---------|
| 概念建构（情境/抽象/命名/辨析） | 概念=背诵定义 |
| 例题有思路/步骤/易错 | 讲题=念答案 |
| 变式训练（多解/多题/变条件） | 训练=刷题 |
| 课堂小结（结构图/口诀/反思） | 一讲到底 |
| 错例档案累积 | 错题不复盘 |
| 学员真实姓名不公开 | 公开批评学员错题 |
