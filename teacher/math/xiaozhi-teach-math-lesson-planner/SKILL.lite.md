# 数学教案设计 · 精简版

> 详细版见 `SKILL.md`。

## 1. 一句话定位

数学不是教学生"记住公式"，而是教学生"用公式思考"——概念建构是核心。

## 2. 工作流

```text
概念建构 → 例题示范 → 变式训练 → 课堂小结 → 错例档案
```

## 3. 概念建构四步

```text
情境（生活/历史/游戏/问题/反例）
抽象（提取共同特征）
命名（给概念命名）
辨析（区别于其他概念）
```

## 4. 例题三要素

```text
思路：怎么想到的
步骤：清晰的步骤
易错：易错点+避免
```

## 5. 变式三类型

```text
一题多解：多解法训练思维灵活性
多题一解：识别共同方法
变条件：改条件/改问题/改情境
```

## 6. 课堂小结三形式

```text
结构图：概念/方法的逻辑结构
口诀：朗朗上口
反思：What / How / Transfer
```

## 7. 错因七分类

```text
概念模糊 / 规则错误 / 审题错误 / 策略错误 /
计算错误 / 粗心大意 / 知识漏洞
```

## 8. 必触发场景

- 老师："这个概念怎么讲"
- 老师："教案怎么写"
- 老师："学员不会做题"
- 老师："数学思维怎么教"
- 老师："变式训练怎么设计"
