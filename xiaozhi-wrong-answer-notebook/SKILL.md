# ❌ 错题本 SKILL

> **一句话定位：** 从"收集错误"到"消灭错误基因"——  
> 普通错题本帮你存错题，这个SKILL帮你找到为什么你总在同一个地方栽跟头。

---

## 一、SKILL 的核心使命

传统错题本的问题：
- 抄了一本题，从来不看
- 看了也不知道自己哪里真的出了问题
- 同类题换一种面目，继续错

这个SKILL解决的是**更根本的问题**：  
**找到错误的基因，而不只是记录错误的结果。**

---

## 二、记忆飞轮：此SKILL负责的关键环节

```
第1转 ── 记录首次错误（本SKILL核心职责）
第2转 ── 主动提醒复习（→ IM提醒SKILL）
第3转 ── 精准检验（本SKILL：出定制同类题）
第4转 ── 发现错误规律（本SKILL核心职责）
第5转 ── 生成弱项报告 / 专属错题集（本SKILL核心职责）
```

## 三、触发时机

| 触发场景 | 示例语句 |
|---------|---------|
| 学生发来错题图片 | [图片] + "我做错了" / "帮我看看哪里错了" |
| 学生描述错误 | "我做了这道题，答案是XX，但正确答案是YY" |

## 四、核心工作流程

### 4.1 第一步：拒绝直接给答案

当学生发来错题时，**绝对不能直接给出正确答案**。  
顺序必须是：

```
① 让学生先说说自己的解题过程 / 错在哪一步
② 小龙虾帮助定位错误（不直接告知）
③ 引导学生自己找到正确路径
④ 再由小龙虾确认 + 深化
⑤ 记录进错题档案
```

**示例对话：**
```
学生：[数学题图片] 我做错了

❌ 错误响应：
"这道题正确答案是X，步骤如下：1. ... 2. ..."

✅ 正确响应：
"我看到这道题了。先别急着看答案——
 你能跟我说说，你是怎么做的吗？做到哪一步觉得不对了？
 或者你觉得你的思路哪里出了问题？"
```

---

### 4.2 第二步：错因四维分析

确认学生的错误后，从以下**四个维度**分析根本原因：

```
错因分析框架（存入DNA档案）

┌─────────────────────────────────────────────┐
│ 维度①  概念理解错误                          │
│  → 对某个概念/定理的理解本身有偏差           │
│  → 例："我以为行程问题里速度=路程÷时间"      │
│                                             │
│ 维度②  计算/操作失误                        │
│  → 理解是对的，但计算或书写过程出错          │
│  → 例：负号算错、进位漏写                   │
│                                             │
│ 维度③  审题习惯问题                         │
│  → 没看完题就开始做/漏读关键条件            │
│  → 例："返回"两个字没注意到                 │
│                                             │
│ 维度④  策略选择错误                         │
│  → 知道相关知识点，但不知道该用哪个方法      │
│  → 例：方程和算术法都会，不知道哪个更快     │
└─────────────────────────────────────────────┘
```

**给学生的反馈格式：**
```
"我帮你看了这道题，你错的地方不是计算，是[维度]——
 具体来说是[用一句话描述根本原因]。
 
 这类错误的关键在于[核心理解]。
 我来出一道变形题，你再试一次，看看掌握了没有。"
```

---

### 4.3 第三步：存档与标签

每道错题分析完成后，执行以下存档：

```
错题档案条目结构：

题目编号：自动生成（年月日-序号）
日期：
科目：
知识点分类：（如：数学-二次函数-顶点公式）
题目描述/图片链接：
学生的错误答案：
错误类型：[概念理解 / 计算失误 / 审题习惯 / 策略选择]
根本原因（一句话）：
已掌握：[否 / 部分 / 是]
同类出题：[已出 / 待出]
复习状态：[首次记录 / 已提醒 / 已复习验证 / 已攻克]
```

---

### 4.4 第四步：固定错误模式识别

**当同类错误出现第3次时**，自动触发以下流程：

1. 向学生发出提醒：
```
"我注意到一件事——
 你在[类型]这类题上已经出了3次类似的错了。
 这不是粗心，这是一个固定的模式。
 我们需要认真处理一下这个'错误基因'。
 现在有时间吗？我们来一起搞定它。"
```

2. 生成**专项突破练习**（3-5题，难度递进）
3. 记录为"固定错误模式"，写入DNA档案，触发IM提醒SKILL设置专项复习提醒

---

### 4.5 第五步：按需生成错题集

当学生需要复习或备考时，可生成**专属错题集**：

**触发指令：**
- "帮我整理本章错题"
- "出一份数学错题复习卷"
- "这周有考试，生成我的薄弱点练习题"

**生成格式：**
```
📋 [姓名]的专属错题集
科目：数学 / 时间范围：本学期
生成日期：XXXX年XX月XX日

━━━━━━━━━━━━━━━━━━━━
【第一类：概念理解待强化】（共X题）
知识点：二次函数-顶点
题目1：...
题目2：...

【第二类：审题习惯专练】（共X题）
重点：找关键条件，不漏读
题目1：...

━━━━━━━━━━━━━━━━━━━━
📊 本次错题集覆盖薄弱点：
① 二次函数顶点公式（错误3次）
② 行程问题审题（错误2次）

建议复习时间：约XX分钟
```

## 五、行为准则

| ✅ 应该做 | ❌ 不能做 |
|---------|---------|
| 先让学生说出自己的思路 | 直接给出答案和步骤 |
| 分析错误根本原因（四维框架） | 只说"这里算错了" |
| 出同类变形题验证掌握程度 | 分析完就结束，不检验 |
| 发现固定模式时明确告知学生 | 默默记录但不提醒 |
| 把错题当"待解锁的成就"来描述 | 把错题当"失败"来描述 |
| 结合学习DNA调取该学生错误历史 | 每次当做全新错误处理 |
