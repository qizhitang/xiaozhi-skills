# 数学错因维度表

> 本文档为 `xiaozhi-error-gene-archive` 的核心参考资源，定义数学学科包的四类错因维度体系、子类型分类、跨维度关联规则与追踪标准。

---

## 一、维度总览

```
数学错因维度体系（四维×多子类型）

维度①  计算失误 ── 操作层错误（思路对，执行错）
维度②  概念模糊 ── 理解层错误（不知道为什么用/怎么用）
维度③  方法用错 ── 策略层错误（知道知识但选错路径）
维度④  读题失误 ── 信息层错误（题目信息获取偏差）

每道错题必须归入且只归入一个维度（主维度）。
如存在跨维度因素，记录为"主维度+次要维度"。
```

---

## 二、维度①：计算失误

**定义：** 解题思路正确，在运算过程中出现技术性错误。

**判断标准：** 如果换个数字/换个具体值，学生能做出同类题，说明思路没问题，只是执行出了偏差。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发学段 | 关联概念混淆 |
|---------|------|---------|---------|------------|
| C01 | 符号错误 | 移项忘变号；负负得正搞反 | 七年级(有理数) | → 概念模糊B01 |
| C02 | 分数运算失误 | 分数加减忘通分；分数乘除混规则 | 七-八年级 | → 概念模糊B03 |
| C03 | 去括号/分配律错误 | 漏乘括号内某项；括号前负号分配错误 | 七年级 | → 概念模糊B02 |
| C04 | 乘方/开方计算错误 | (-2)²算成-4；√9算成3但√4算成±2 | 七-八年级 | → 概念模糊B04 |
| C05 | 解方程步骤遗漏 | 两边同时除时漏了检验；代入时忘写某步 | 八-九年级 | — |
| C06 | 几何计算粗心 | 角度加算错误；面积公式代入数值错 | 八-九年级 | → 概念模糊B07 |
| C07 | 数据抄写/复制错误 | 把题目数字抄错；从上一行抄到下一行出错 | 全学段 | — |
| C08 | 进位/退位失误 | 大数加减中进退位出错 | 小学-七年级 | — |

**严重程度分级：**

```
轻度（偶发）：C07、C08 类型 → 粗心层面
  → 处理：提醒注意，不启动专项训练

中度（模式化）：C01、C02、C03、C05 类型 → 同一操作反复出错
  → 处理：专项计算步骤训练
  → 目标：建立"自检清单"（每步完成后检查符号、分母、括号）

重度（顽固）：计算失误+概念模糊交叉 → C01伴随B01
  → 处理：需先解决概念问题，再治计算习惯
  → 符号错误频繁出现 → 可能不是粗心，而是对"负数运算规则"概念模糊
```

**根治方法映射：**

```
C01 符号错误 → 专项训练：每步完成后回查符号
  → 辅助工具："符号自检三步法"（移项→变号？；负号→分配？；结果→正负？）

C02 分数运算 → 专项训练：通分前先写分母最小公倍数
  → 辅助工具："分数运算四步检查"

C03 去括号 → 专项训练：括号前符号标红后逐项分配
  → 辅助工具："括号展开清单"

C05 解方程遗漏 → 专项训练：关键步骤标注"★"提醒自己
  → 辅助工具："方程解题步骤清单"
```

## 三、维度②：概念模糊

**定义：** 对某个定义、定理、公式的理解有偏差，不知道什么情况下该用，或把相似概念混用。

**判断标准：** 学生说"我知道这个定义但不知道什么时候用它"，或把两个相似概念搞混。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发知识点 | 关联参考 |
|---------|------|---------|-----------|---------|
| B01 | 符号规则理解偏差 | 知道"负负得正"但不知道什么时候用 | 有理数运算 | → concept-confusion-map §一.1 |
| B02 | 运算优先级混淆 | 先乘除后加减，但遇到括号混乱 | 整式运算 | → concept-confusion-map §一.1 |
| B03 | 分式vs整式概念混淆 | 不知道什么条件下是分式 | 多项式与分式 | → concept-confusion-map §一.3 |
| B04 | 乘方vs乘法混淆 | (-2)²和-2²搞不清区别 | 乘方运算 | → concept-confusion-map §一.1 |
| B05 | 方程解vs函数零点 | 解方程得到数值 vs 函数图像与x轴交点 | 一次函数 | → concept-confusion-map §一.2 |
| B06 | 正比例vs一次函数 | y=kx 和 y=kx+b 的关系 | 函数概念 | → concept-confusion-map §一.2 |
| B07 | 几何判定条件混淆 | SSA不成立但SAS成立；全等vs相似 | 三角形 | → concept-confusion-map §二.1 |
| B08 | 统计概念混淆 | 平均数vs中位数vs众数的适用场景 | 统计与概率 | → concept-confusion-map §三 |
| B09 | 绝对值理解偏差 | |x|是距离而非"去掉负号" | 有理数 | → concept-confusion-map §一.1 |

**严重程度分级：**

```
轻度：单一知识点概念偏差 → 可通过类比和追问一次解决
  → 处理：用概念解释器SKILL一次追问重建理解

中度：跨知识点概念混淆 → B05/B06/B07类型（两个概念搞混）
  → 处理：概念对照表 + 追问 + 3道对比题
  → 工具：concept-confusion-map.md 对照追问

重度：根基概念缺失 → 影响多个后续知识点
  → 如：不等式概念建立在方程概念之上，如果方程概念模糊，
    不等式必定反复出错
  → 处理：溯源补充前置知识，重建理解链
```

**根治方法映射：**

```
B类错误通用根治路径：
  ① 追问定位：用苏格拉底五问链第1问确认偏差方向
  ② 类比重建：用概念解释器SKILL提供生活类比
  ③ 对照区分：用concept-confusion-map做相似概念对比
  ④ 应用验证：出3道纯概念应用题确认理解
  ⑤ 迁移确认：出1道变形题确认不是只会原题

关键区分原则：
  → 如果学生说"我懂了"但同类题还是错 → B类（概念模糊）
    而不是C类（计算失误）
  → 区分标准：纯净版题（只含该知识点）做对了吗？
    做对了 → C类（计算失误，干扰条件下出错）
    做错了 → B类（概念本身有偏差）
```

## 四、维度③：方法用错

**定义：** 知道相关知识点，但选错了解题策略——思路对但绕远路，或题目变形后不会调整方法。

**判断标准：** 学生能说出正确的知识点，但选了错误的解题路径，或用了不适合当前题型的方法。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发题型 | 核心问题 |
|---------|------|---------|---------|---------|
| M01 | 题型识别错误 | 把证明题当计算题做；把应用题当纯代数题做 | 综合 | 不会从题目特征判断该用什么方法 |
| M02 | 方法选择不当 | 该用配方法却用公式法；该用辅助线却硬算 | 函数/几何 | 方法库丰富但选择策略缺失 |
| M03 | 方法迁移失败 | 原题会做但变形后不会（方法绑定在原题形式上） | 变式题 | 方法记忆而非方法理解 |
| M04 | 过度复杂化 | 用了最复杂的方法而非最简路径 | 综合 | 缺少"最优路径"意识 |
| M05 | 中途切换方法 | 一半用方法A，一半换成方法B，两套混搭 | 综合 | 缺少全程规划意识 |
| M06 | 缺少辅助工具 | 不画图/不列表/不设方程，全靠心算推理 | 应用题/几何 | 不习惯使用数学工具 |

**严重程度分级：**

```
轻度：M04、M06 → 方法选择不够优，但最终能做出
  → 处理：指出更优路径，不启动专项训练

中度：M01、M02、M05 → 题型识别或方法选择有系统性偏差
  → 处理：题型识别训练 + "哪类题用哪类方法"对应表
  → 工具：CLAW模板③（思路卡壳类）引导正确方法选择

重度：M03 → 方法迁移失败，只会"这道题"而非"这类题"
  → 处理：苏格拉底五问链第5问（个性化沉淀）
  → 目标：建立题型识别特征 → 方法映射
```

**根治方法映射：**

```
M01/M02 题型识别训练：
  Step 1：给5道不同题型，让学生先判断"这是什么类型"
  Step 2：确认题型后，再选择方法（不是直接开始做）
  Step 3：建立个人"题型→方法"映射表

M03 方法迁移训练：
  → 苏格拉底五问链完整版（特别是第2问假设检验+第5问个性化沉淀）
  → 每次做完题后，让学生总结：这个方法的识别特征是什么？
  → 出3道同方法但不同形式的变式题

M06 辅助工具意识训练：
  → 每次做题前，先选择工具：画图？列表？设方程？
  → 不同工具适合不同题型（建立对应关系）
```

## 五、维度④：读题失误

**定义：** 题意理解偏差，漏读或误读条件，导致解题方向从一开始就是错的。

**判断标准：** 看了三遍题目但还是漏了关键信息；或把"不少于"理解成"等于"。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发陷阱词 | 关联参考 |
|---------|------|---------|-----------|---------|
| R01 | 漏读关键词 | 题目说"整数解"，学生算了分数解 | "整数解""正数解""最大值" | → reading-habits §三 |
| R02 | 误读数量关系 | 把"A是B的3倍"看成"B是A的3倍" | "A的N倍""比A多/少N""占A的N%" | → reading-habits §三 |
| R03 | 忽略特殊条件 | 题目说"x≠0"，学生代入了0 | "x≠0""均为整数""互不相等" | → reading-habits §二 |
| R04 | 多步题只解第一问 | 有(1)(2)两问，只做了(1) | 题目含多问标记 | → reading-habits §四 |
| R05 | 图形信息遗漏 | 图中有标注但学生没注意到 | 图形中隐含角度/长度标注 | → reading-habits §二 |
| R06 | 情境误解 | "相向"理解成"同向""返回"理解成"继续" | "相向""返回""往返""顺流/逆流" | → reading-habits §三 |

**严重程度分级：**

```
轻度：R01、R04 → 偶尔漏看，注意力分配问题
  → 处理：读题清单法（30秒检查清单）

中度：R02、R06 → 数量关系/情境词系统性误读
  → 处理：陷阱词专项训练 + 数量关系翻译练习
  → 工具：reading-habits.md 陷阱词速查表

重度：R03、R05 → 条件意识缺失（习惯性忽略限制条件和图形标注）
  → 处理：强制读题流程（先找限制条件→再找图形标注→再解题）
  → 建立"条件提取清单"习惯
```

**根治方法映射：**

```
R类错误通用根治路径：
  ① 读题清单法：每道题开始前30秒完成5项检查
  ② 关键词高亮练习：用笔圈出数量关系词和限制条件
  ③ 条件翻译练习：把"不少于""比A多N"等翻译为数学表达
  ④ 验证回代习惯：做完后把答案代回题目，检查所有条件是否满足

关键区分原则：
  → 如果学生的解题过程完全合理但答案不对 → R类（读题失误）
  → 如果解题过程本身就有问题 → B类/M类（概念模糊/方法用错）
  → 如果过程对了但某一步算错 → C类（计算失误）
```

## 六、跨维度关联规则

```
维度间常见交叉：

C01(符号错误) + B01(符号规则理解偏差)
  → 判断：纯净版题做对了吗？
    做对了 → C类为主（干扰条件下的执行失误）
    做错了 → B类为主（概念本身理解偏差，不是粗心）

M03(方法迁移失败) + B类(概念模糊)
  → 判断：换一道同知识点但不同题型的题，会做吗？
    会做 → M03为主（方法绑定在题型形式上）
    不会做 → B类为主（概念理解不够导致方法无法迁移）

R02(误读数量关系) + M01(题型识别错误)
  → 判断：题目条件理解正确后，会做吗？
    会做 → R类为主（信息获取层面的问题）
    不会做 → M类为主（即使读对了题也选错方法）

跨维度记录格式：
  主维度：[维度ID+子类型ID]
  次要维度：[维度ID+子类型ID]（如有）
  交叉判断依据：[纯净版题结果/条件修正后结果]

优先处理顺序：
  → B类(概念) > R类(读题) > M类(方法) > C类(计算)
  → 先解决理解层(B) → 再解决信息层(R) → 再解决策略层(M) → 最后解决操作层(C)
  → 概念模糊是底层原因，其他维度的问题往往是概念问题的表面表现
```

## 七、顽固弱项追踪标准

```
同一子类型（相同维度+子类型ID+相同知识点标签）出现次数追踪：

出现1次 → 标记"偶发"，低优先级
出现2次 → 标记"初步弱项"，进入观察期
出现≥3次 → 标记"顽固弱项"，启动专项突破流程

顽固弱项突破流程：
  Step 1：纯净版题测试（只含该知识点，排除干扰）
    → 做对 → 干扰条件下识别问题 → 出3道递进干扰题
    → 做错 → 概念/方法根因问题 → 联动概念解释器重建
  Step 2：专项训练（针对子类型的根治方法）
  Step 3：验证锁定（次日+3天后各验证一次）
    → 连续2次做对 → 标记"攻克（待巩固）"
    → 连续3次做对 → 标记"已攻克"

攻克后复发处理：
  → 攻克后再次出现同子类型错误 → 降级为"突破中"
  → 连续2次复发 → 重新启动专项突破流程

月报统计维度：
  → 四维度占比饼图
  → 各维度内子类型占比排序
  → 顽固弱项清单（含出现次数+当前状态）
  → 跨维度交叉分析（如有）
```

## 八、与通用错题本的协作边界

```
本维度表是数学专属扩展，与通用错题本SKILL的关系：

通用错题本职责：
  → 记录错题表面信息（题目、答案、日期）
  → 通用错误分类（使用 error-analysis-framework.md 中的通用分类）
  → 跨科目统一管理

数学错误基因档案职责（本维度表）：
  → 数学专属四维度深度分类
  → 子类型精确定位（C01/B01/M01/R01等）
  → 跨维度关联分析
  → 顽固弱项专项突破

数据流转规则：
  → 通用错题本记录 → 推送到数学错误基因档案 → 使用本维度表进行深度分类
  → 数学错误基因档案分析结果 → 可回写到通用错题本的"根因分析"字段
  → 不产生重复记录：通用层记表面，数学层记根因
```
