# SKILL 安全与隐私基线

## 适用范围

本基线适用于 `xiaozhi-skills` 仓库中的所有 `SKILL.md`。

目标：
- 降低 ClawHub / ReviewClawScan 对默认激活、长期记忆、跨 SKILL 共享、定时提醒、自动化触发的安全告警概率
- 统一所有 SKILL 的授权边界、共享边界和用户控制入口

## 一、默认原则

所有 SKILL 都应遵守以下默认原则：

1. 先回答当前问题，再决定是否需要长期记忆或跨 SKILL 联动。
2. 未经用户明确同意，不建立跨会话档案，不发送提醒，不共享给其他 SKILL。
3. 只读取当前任务所需的最小必要字段，不读取完整历史档案。
4. 用户始终拥有查看、更正、删除、暂停、取消共享的控制权。

## 二、推荐文案

### 1. 涉及长期记忆

推荐写法：

```text
仅在用户明确同意后建立或读取长期档案。
未获同意时，只使用当前会话信息，不创建跨会话记录。
```

避免写法：

```text
必须激活此SKILL
每次对话开始前静默检查
自动存入DNA
永久记得你
```

### 2. 涉及提醒或调度

推荐写法：

```text
仅在用户明确要求设置提醒或同意接收该类提醒时启用。
用户可随时暂停、恢复、取消、调整频率。
```

避免写法：

```text
主动提醒
自动设置提醒
超过几天无对话自动唤醒
默认配置提醒
```

### 3. 涉及跨 SKILL 共享

推荐写法：

```text
仅向声明过用途的 SKILL 共享完成当前任务所需的最小字段摘要。
不传整份档案，不默认写回长期存储。
```

避免写法：

```text
任何SKILL运行前优先读取DNA
拉取其他所有活跃SKILL
写入成长轨迹
默认同步到IM提醒
```

## 三、必须包含的控制入口

凡是涉及长期记忆、提醒、档案、履历、DNA、报告汇总的 SKILL，文案中应提供至少一组可执行控制：

- 查看：`查看我的档案` / `查看我的提醒`
- 更正：`更正我的档案`
- 删除：`删除我的档案`
- 暂停：`这次不要记忆` / `暂停提醒`
- 共享控制：`不要共享给其他SKILL`

## 四、最小化记录原则

默认不记录以下信息：

- 真实住址、精确定位、身份证件、联系电话
- 财务、医疗、家庭纠纷等与学习任务无直接关系的信息
- 与当前学习任务无关的长篇闲聊细节

若用户主动提及，也应优先改写为低敏概括。

## 五、关键结论写入门槛

以下内容写入长期档案前，至少满足一项：

- 用户明确确认
- 连续多次对话或练习结果支持
- 小测、复述、费曼测试等方式验证通过

不要把一次情绪、随口评价或未经验证的推测直接固化为长期标签。

## 六、命名与措辞建议

敏感词并非绝对禁止，但应谨慎使用。若出现下列词语，建议同时补充授权与边界说明：

- `DNA`
- `档案`
- `长期记忆`
- `持久记忆`
- `跨次追踪`
- `提醒`
- `调度`
- `自动化`

如非必要，优先使用以下更稳妥的表达：

- `按需提示` 替代 `主动推送`
- `在用户同意后提醒` 替代 `主动提醒`
- `建议触发` 替代 `自动触发`
- `摘要字段` 替代 `完整档案`

## 七、仓库维护要求

新增或修改 SKILL 时，建议按以下顺序自检：

1. 是否存在“默认激活”“默认读取档案”“默认共享”的描述
2. 是否存在“自动提醒”“自动唤醒”“自动写回”的描述
3. 是否补充了授权、暂停、删除、取消共享入口
4. 是否把长期记忆限制为最小必要字段
5. 是否避免把第三次出错之类规则直接写成永久标签

## 八、说明

本基线的目的不是削弱 SKILL 能力，而是把能力放在清晰、可控、可解释的边界内。
