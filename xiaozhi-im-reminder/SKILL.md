# ⏰ IM智能提醒 SKILL

> **一句话定位：** 不是你去找小龙虾，而是它在你快忘、快拖延、快中断的时候来找你。

## ⚠️ 技术实现边界声明

> **关于“定时主动提醒”机制：** 本模块中涉及的主动提醒调度，并非 LLM 自身具备的自主计时能力。它依托于如 QClaw 等定制化 OpenClaw 平台的「定时任务机制」与「自动化工作流引擎」在指定时间唤醒大模型。同时，消息的主动推送泛化支持任何已接入 OpenClaw 平台的 IM 软件（如微信、QQ等），并非针对单一软件定制。

---

## 一、SKILL 的核心使命

原版使命是把遗忘曲线变成复习曲线。  
v1.1在此基础上新增两类提醒：
- 任务执行提醒
- 探索成长提醒

所以它现在负责三件事：

```text
1. 复习提醒
2. 行动提醒
3. 回访确认提醒
```

---

## 二、触发时机

| 触发场景 | 示例语句 | 处理方式 |
|---------|---------|---------|
| 学完新内容主动汇报 | “我今天学了二次函数顶点公式” | 自动设置复习提醒序列 |
| 明确要求设置提醒 | “帮我明天提醒我复习这个” | 立即确认并设置 |
| 错题分析完成后 | 错题本分析结束 | 自动设置强化提醒 |
| 新建学习任务后 | “我这周每天晚饭后背20个词” | 建立任务提醒 |
| 兴趣探索启动后 | “这周我要完成第3杯咖啡聊天” | 建立探索提醒 |
| 每日时间记录回访 | “今天我会记录时间花在哪了” | 晚间确认提醒 |
| 长时间未使用 | 超过3天无对话 | 发送温柔唤醒提醒 |

## 三、标准复习提醒序列

### 3.1 普通知识点

```text
Day 1  → 第一次复习提醒
Day 3  → 第二次复习提醒
Day 7  → 第三次复习提醒
Day 14 → 第四次复习提醒
Day 30 → 巩固验证提醒
```

调整规则：
- 学生确认“已记住”并验证通过，间隔可以拉长
- 学生确认“忘了”或答错，间隔缩短并回退阶段

### 3.2 错题强化序列

```text
Day 1  → 检查根因是否已理解
Day 2  → 出一道变形题
Day 5  → 再测一次
Day 10 → 综合检验
Day 21 → 最终确认
```

若属于固定错误模式，优先级提升。

## 四、新增触发场景一：计划任务定时提醒

来源：与`30天学习计划制定师`联动。

### 4.1 适用场景

```text
1. 学生刚制定了未来7天 / 30天任务表
2. 某任务有明确执行时间
3. 某任务连续拖延，需要回访推进
```

### 4.2 提醒内容要求

计划提醒不能只说“记得做”，而要说清：
- 今天要做什么
- 预计用多久
- 做完后回复什么关键词

推荐模板：

```text
"提醒一下，今天你的计划任务是：
[任务名称]
预计耗时：[X]分钟

做完回复我‘完成’，我会帮你记录进度。
如果今天做不了，也直接告诉我，我们一起调整。"
```

## 五、新增触发场景二：52杯咖啡探索提醒

来源：与`兴趣成长探索计划`联动。

### 5.1 作用

用于提醒学生完成兴趣探索任务，而不是把“想探索”停留在口头上。

适用内容：
- 本周第几杯咖啡聊天
- 本周要接触的行业 / 角色 / 主题
- 本次探索后的简短回收记录

### 5.2 推荐模板

```text
"这周的探索任务别忘了：
去完成你的第[ N ]杯咖啡。

目标不是表现自己，而是带着好奇问3个问题：
1. 你平时最真实的一天怎么过？
2. 这个方向最难的地方是什么？
3. 你为什么还愿意继续做？

聊完回来告诉我，你的感觉更接近：
没感觉 / 有点意思 / 想继续了解。"
```

## 六、新增触发场景三：时间销行账每日确认提醒

来源：与`时间与专注力教练`联动。

### 6.1 目标

不是督促学生“更努力”，而是帮助他看清今天时间到底流向了哪里。

### 6.2 触发方式

当学生当天开始记录时间使用情况，或处于专注调整期时，默认可设置晚间确认提醒。

推荐模板：

```text
"今天快结束了，来做个30秒确认：

你今天的时间大概花在了哪三件事上？
最值的一段时间是什么？
最容易滑走的一段时间是什么？

不用写很长，给我关键词就行。"
```

作用：
- 让时间记录形成闭环
- 给周复盘提供真实材料
- 帮助学生发现分心高发时段

## 七、提醒消息格式规范

所有提醒都必须满足三个原则：

```text
1. 有内容，不是空提示
2. 有动作，告诉学生该怎么回应
3. 有温度，不制造压力
```

示例：

```text
[提醒] 昨天学的[知识点]，今天来巩固一下。
先试一道小题：[题目]
如果你还不确定，直接回复“再讲一遍”。
```

---

## 八、学生响应处理

| 学生回复 | 小龙虾处理 |
|---------|---------|
| 答对了 / 完成了 | 记录完成并拉长间隔或进入下一阶段 |
| 答错了 / 没完成 | 不批评，缩短间隔或重新拆任务 |
| “记得了” | 仍需快速验证 |
| “忘了” | 重新讲解并回退提醒阶段 |
| “稍后” | 设置延后提醒 |
| 无回应 | 仅做一次温柔补发，不连环轰炸 |

---

## 九、提醒数量管理

为了避免提醒疲劳，遵守以下规则：

```text
普通日：最多3条
考前冲刺：最多5条
夜间免打扰：22:00-8:00 默认不发
优先顺序：
1. 考前关键提醒
2. 固定错误模式提醒
3. 计划任务提醒
4. 探索提醒
5. 常规复习提醒
6. 每日确认提醒
```

发送时间优先参考学习DNA中的历史活跃时段。

## 十、复习与行动队列可视化

当学生问“我现在有哪些要处理的？”时，输出统一队列：

```text
📋 今日队列

🔴 高优先级
  固定错误模式复测 / 今天必须完成的计划任务

🟡 今日到期
  某知识点Day 3复习 / 某探索任务回访

🟢 本周内
  某知识点Day 14复习 / 每日时间确认
```

让学生按优先级处理，而不是同时记很多事。

```text
    ──→ 每周学习复盘（完成率与响应率数据）

```
