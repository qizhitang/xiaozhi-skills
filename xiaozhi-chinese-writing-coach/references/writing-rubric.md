# 三文体评分维度与追问问题库

---

## 一、三维批改法评分维度详解

### 议论文三维批改

**论点层**
| 检验问题 | 常见问题 | 追问方式 |
|---------|---------|---------|
| 论点是否清晰 | 论点太宽泛，人人都能说 | "用一句话说你的观点，不能超过20个字" |
| 论点是否有边界 | 说了"所有情况"其实只能说"部分情况" | "这个观点在什么情况下不成立？" |
| 论点是否有新意 | 论点是套话，毫无个人立场 | "这个观点是你自己真的相信的吗？有没有你自己的经历支持它？" |

**论据层**
| 检验问题 | 常见问题 | 追问方式 |
|---------|---------|---------|
| 论据是否有说服力 | 举了名人但不具体 | "这个例子能说明什么？你能把它说得更具体吗？" |
| 论据是否多样 | 只用同类型例子 | "你用的例子都是[类型]，有没有不同类型的证明？" |
| 论据与论点的关联 | 例子没有指向论点 | "这个例子怎么支持你的论点？中间的逻辑是什么？" |

**语言层**
| 检验问题 | 常见问题 | 追问方式 |
|---------|---------|---------|
| 有无低效连词 | "然后""还有""另外"频繁出现 | 直接告知：出现N次，建议替换为[逻辑连词] |
| 有无套话 | "众所周知""不言而喻""毋庸置疑" | 直接指出，建议删除直接进入内容 |
| 表达是否精准 | 意思对但表达绕 | "这句话能不能用更短的方式说？核心意思是什么？" |

---

### 记叙文三维批改

**情感层**（替换议论文的"论点层"）
| 检验问题 | 常见问题 | 追问方式 |
|---------|---------|---------|
| 有无真实情感 | 只有事件，没有感受 | "读到这段，我感觉不到你当时的情绪，你当时是什么感受？" |
| 情感是否具体 | 情感笼统（"我很开心/难过"） | "你说'难过'——是什么样的难过？心里有什么感觉？" |
| 情感是否自然 | 情感过度，失真 | "这里的感受读起来有点夸张，还是你真实的感受吗？" |

**细节层**（替换"论据层"）
| 检验问题 | 常见问题 | 追问方式 |
|---------|---------|---------|
| 细节是否具体 | 场景描写太笼统 | "这个场景，你能说说当时的环境、声音、气味吗？" |
| 动作描写是否到位 | 只写了做了什么，没写怎么做的 | "这个动作，你能慢镜头描述一下具体过程吗？" |
| 对话是否自然 | 对话像书面语，不像真实说话 | "这段对话，当时真的是这么说的吗？还是更像这样说的？" |

**结构层**（替换"语言层"）
| 检验问题 | 常见问题 |
|---------|---------|
| 开头是否吸引人 | 从太远的背景开始，迟迟不进入正题 |
| 详略是否得当 | 重要场景写得很简短，次要场景却很详细 |
| 结尾是否有力 | 结尾是套话总结，没有回应开头或升华 |

---

### 说明文三维批改

**逻辑顺序层**
| 检验问题 | 常见问题 |
|---------|---------|
| 说明顺序是否一贯 | 时间/空间/逻辑顺序中途混用 |
| 段与段之间是否有过渡 | 各段之间跳跃，读者跟不上 |
| 说明层次是否清晰 | 总分结构不清，读完不知道说了什么 |

**读者视角层**
| 检验问题 | 追问方式 |
|---------|---------|
| 有无知识假设 | "这里你说了[专业词]，读者可能不知道是什么意思，要不要解释？" |
| 说明是否完整 | "读完这段，读者能知道[目标知识]吗？还是还会有疑问？" |

**说明方法层**
| 说明方法 | 检验是否运用得当 |
|---------|---------------|
| 举例子 | 例子是否典型，是否够具体 |
| 列数字 | 数字是否有出处，是否真实准确 |
| 打比方 | 比喻是否恰当，有没有误导 |
| 作比较 | 对比是否清晰，两者的差异是否被说明 |
| 下定义 | 定义是否准确，是否覆盖了所有情况 |

## 二、苏格拉底四连追问话术详细版

### 第一连：澄清论点

**适用场景：** 论点过于宽泛、模糊、或有歧义

```
基础版："你的核心观点用一句话概括是什么？"

加强版（当论点有边界问题时）：
"你说[论点]——这个论点是说'在所有情况下'都成立，
 还是'在特定条件下'成立？
 比如，有没有什么情况下，这个论点不成立？"

追逼版（当学生绕圈子时）：
"如果让你把这篇文章的核心观点写在一张便利贴上，
 你会写什么？就是那一句话。"
```

### 第二连：检验证据

**适用场景：** 论据缺乏、空洞、或与论点脱节

```
基础版："你有什么具体事实或数据来支持这个观点？"

深挖版（当论据是名人例子时）：
"你用了[名人]的例子——能说得更具体吗？
 他/她具体做了什么，怎么证明了你的观点？"

个人化版（当论据缺乏个人经历时）：
"有没有你自己亲身经历过的事情，可以支持这个观点？
 个人经历往往比名人故事更有说服力。"
```

### 第三连：引入对立视角

**适用场景：** 论证单方面，缺乏深度

```
基础版："有没有人会反对你？他们会怎么说？"

引导版（当学生想不到反驳时）：
"我来扮演反对你的人——我说：'[你的论点]这个观点根本站不住脚，
 因为[编一个合理的对立理由]。' 你怎么回应我？"

升华版（当学生回应了反驳）：
"你的回应很好。那你能把这个'对立观点+你的回应'
 整合进你的论证里吗？这叫驳论，会让你的文章更有力。"
```

### 第四连：推向结论

**适用场景：** 经过前三连追问后，论点需要升级

```
基础版："综合你的论据和反驳，你的最终观点是什么？"

对比版（帮学生看到进步）：
"你最开始的论点是'[原论点]'，
 经过刚才的讨论，你现在的论点是'[新论点]'。
 你觉得哪个更有说服力？为什么？"
```

## 三、AI辩论赛详细操作手册

### 三轮辩论完整脚本

**赛前宣告（每次辩论前都要说）：**
```
"辩论规则说清楚——
 我会认真地站在[对立立场]，找你论证里的漏洞，
 不会故意让你，也不会过分刁难你。
 每轮你说完，我来反驳，然后你再回应，共三轮。
 辩完之后，我会切换成教练视角，告诉你哪里最薄弱。
 好，你开始第一轮陈述。"
```

**教练复盘格式：**
```
"辩论结束，我来做教练点评——

 整体评价：[一句话总体评价]

 你最强的一个论点：[指出最有力的那个]
 因为它[说明为什么有力]

 你最薄弱的一个环节：[具体指出]
 原因是[解释根本原因]
 下次遇到这个反驳，可以从[方向]来应对

 如果你要把这场辩论整理成一篇议论文，
 最值得保留的素材是：[指出1-2个]"
```
