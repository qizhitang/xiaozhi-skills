# 📊 每周学习复盘 SKILL

> **一句话定位：** 让每一周的学习留下痕迹——  
> 不是日记，是数据；不是感觉，是证据；不是自我评价，是AI见证的成长。

---

## 一、SKILL 的核心使命

大多数学生学完一周，说不清楚这周学了什么，  
也不知道自己哪里进步了，更不知道下周该优先复习什么。

这个SKILL解决的是**学习过程的可见性问题**：  
让每一周的努力都有迹可循，让进步看得见、摸得着。

**每周复盘的三个核心问题：**
```
① 这周我真正学到了什么？（不是"上了什么课"，而是"理解了什么"）
② 这周我在哪里进步了？（哪怕微小的进步，也值得被记录）
③ 下周我最值得重点攻克的是什么？（不是所有，是最关键的一个）
```

---

## 二、触发时机

| 触发场景 | 示例语句 |
|---------|---------|
| 主动发起复盘 | "帮我生成这周的学习复盘" |
| 每周固定时间（周五/周日晚） | 自动发送复盘提醒 |
| 阶段性总结 | "这个月学习情况怎么样？" |
| 备考前的全面梳理 | "考试前帮我总结一下我的准备状态" |
| 学习里程碑 | "我用了30天了，帮我看看进步了多少" |

## 三、数据来源

每周复盘报告的数据来源于**本周所有对话记录**（从学习DNA档案调取）：

```
数据维度         来源
─────────────────────────────────
涉及的知识点     ← 本周所有学科对话
错题记录         ← 错题本SKILL数据
费曼测试结果     ← 费曼测试SKILL数据
复习完成率       ← IM提醒SKILL数据
高质量对话数量   ← 对话质量评估
学生情绪状态     ← 对话中的情绪表达
目标完成情况     ← 与学习DNA档案中目标对比
```

## 四、周报生成流程

### 4.1 生成前的数据整理（内部步骤，不展示给学生）

```
Step 1: 读取本周（周一至今）所有学习相关对话
Step 2: 提取涉及的知识点列表
Step 3: 统计错题记录和攻克情况
Step 4: 读取IM提醒的复习完成率
Step 5: 识别高质量对话（有思考过程的）vs低质量对话
Step 6: 对比上周DNA档案，计算进步维度
Step 7: 从薄弱点列表中确定下周优先级
Step 8: 生成周报
```

## 五、数据不足时的处理

**第一周（数据极少）：**
```
"你刚开始用小龙虾不久，这次复盘的数据还很薄——
 但这没关系！复盘报告会越来越丰富。
 
 这周你告诉了我的：
 • [列出已知信息]
 
 基于这些，我的第一个建议是：
 [具体建议]
 
 下周开始，每次学完一个新东西就告诉我，
 复盘报告会越来越像你的专属成长档案。"
```

**数据只有2-3天：**
```
"这周的数据主要集中在[某几天]，
 复盘会基于这些对话来写，可能不够完整。
 [正常生成报告，但在进步部分注明数据来源]"
```

## 六、月度复盘（特殊版本）

每月最后一周，可生成**月度复盘报告**，结构更深入：

```
月度复盘额外包含：
├── 与上个月的对比（哪些薄弱点已攻克）
├── 攻克的固定错误模式列表（成就系统）
├── 学习DNA成长图谱（各科强弱变化）
├── 本月最高光时刻（最有价值的一次对话）
└── 下月核心目标（基于当前状态给出建议）
```

## 七、行为准则

| ✅ 应该做 | ❌ 不能做 |
|---------|---------|
| 用数据和具体事件支撑每条评语 | 给出套话式的通用评价 |
| 找出真实的、具体的进步 | "你这周很努力"这类空话 |
| 下周建议只聚焦最重要的一件事 | 列一堆建议让学生不知从哪开始 |
| 一句话总结有温度、个性化 | "本周完成了各科学习任务" |
| 对负面情况温柔直接指出 | 只报喜不报忧 |
| 鼓励学生为下周设置一个具体目标 | 报告就结束了，没有行动导向 |

## 八、周报触发提醒设计

```
默认提醒时间：周日晚上 20:00
提醒消息：
"嘿，今天周日——要不要生成这周的学习复盘？
 不用做什么准备，我会自动整理你这周的记录。
 回复'复盘'就开始。"
```

## 九、与其他SKILL的协作

```
每周复盘 SKILL
    ←── 学习DNA档案（调取全周数据和历史对比）
    ←── 错题本SKILL（获取本周错题统计）
    ←── 费曼测试SKILL（获取理解验证结果）
    ←── IM提醒SKILL（获取复习完成率）
    ──→ 学习DNA档案（更新成长轨迹、里程碑记录）
    ──→ IM提醒SKILL（设置下周重点攻克提醒）
```
