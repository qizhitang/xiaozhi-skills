# 📊 每周学习复盘 SKILL

> **一句话定位：** 让每一周的学习留下可追踪的证据，而不是模糊的感觉。

---

## 一、SKILL 的核心使命

很多学生一周学完后，说不清楚：
- 这周真正学到了什么
- 哪些地方真的进步了
- 下周最该优先攻克什么

这个SKILL的任务，是把零散的学习记录整理成**有结论、有证据、有行动建议**的周报。

⚠️ **【架构定位声明】：本SKILL专职负责“每周（Weekly）”维度的报告生成与自我复盘。**
- 本SKILL不涉及任何跨SKILL的触发与调度逻辑（由“联动协调器”负责）。
- 本SKILL不生成月报（月度报告统一由“联动协调器”生成全景月报）。

---

## 二、三维度复盘框架

v1.1将原来的基础复盘，升级为三个核心维度：

```text
维度一：学了什么
  看知识点覆盖、理解深度、练习类型

维度二：哪里变强了
  看攻克、改善、连续性、稳定性

维度三：下一步该干什么
  看优先级，而不是堆建议
```

这三个维度最终展开成六模块周报。

---

## 三、触发时机

| 触发场景 | 示例语句 |
|---------|---------|
| 主动发起复盘 | “帮我生成这周的学习复盘” |
| 每周固定时间 | 周五或周日晚，经用户同意后提醒 |
| 阶段性总结 | “这个月学习情况怎么样？” |
| 备考前梳理 | “考试前帮我总结准备状态” |
| 学习里程碑 | “我用了30天了，帮我看看进步了多少” |

---

## 四、数据来源

每周复盘报告的数据来自本周的全部学习活动：

```text
知识点记录       ← 本周所有学习相关对话
错题统计         ← 错题本
理解深度         ← 费曼测试
提醒完成率       ← IM智能提醒
计划执行情况     ← 30天学习计划制定师
专注表现         ← 时间与专注力教练
兴趣探索记录     ← 兴趣成长探索计划
成长里程碑       ← 学习DNA档案
```

## 五、六模块周报结构

v1.1将原来的周报扩展为六模块，必须按此结构输出。

### 5.1 模块一：本周一句话总结

必须有温度，但不能空泛。  
要求用真实事件概括本周最重要的变化。

### 5.2 模块二：知识点地图

输出本周：
- 新接触知识点
- 重点复习知识点
- 已达到“真正掌握”的知识点

### 5.3 模块三：错题与薄弱点

输出：
- 本周错题总数
- 已攻克 / 分析中 / 固定模式数量
- 最值得关注的一类错误
- 是否触发弱项预警

### 5.4 模块四：理解深度与成长证据

输出：
- 本周费曼测试结果
- 新增“真正掌握”知识点
- 哪些地方从“会复述”提升到“会解释”或“真正掌握”

### 5.5 模块五：执行力与完成率

输出：
- 复习提醒完成率
- 计划执行率
- 专注时间或专注稳定性变化

### 5.6 模块六：下周重点攻克

只能选一个最值得优先处理的方向，并给出3条以内的具体行动建议。

## 六、学习存折量化体系

v1.1新增“学习存折”概念，用来把本周投入与产出量化呈现。

推荐指标：

```text
知识存款：本周新增掌握知识点数量
错题回收：本周已攻克错题 / 已分析错题
理解升级：费曼测试从低级到高级的提升数
执行存款：提醒与计划完成率
成长成就：新增里程碑数量
```

展示原则：
- 用于帮助学生看见积累，不是制造焦虑
- 允许数据不高，但必须解释真实含义
- 数据后面要跟一句解释，不只给数字

---

## 七、苏格拉底自我复盘七问

在给下周建议前，优先引导学生做深度自省。  
不要七问全砸给学生，可择其重点使用。

```text
1. 这周我最真实的进步是什么？
2. 这个进步是怎么发生的？
3. 我这周最常见的卡点是什么？
4. 这个卡点背后更深的原因可能是什么？
5. 哪件事其实做得比我想象中好？
6. 如果下周只能优先改一件事，应该是什么？
7. 我愿意为这件事采取的最小行动是什么？
```

作用：
- 防止周报沦为“AI替我总结”
- 让学生成为复盘的共同参与者
- 提升行动承诺度

---

## 八、成长曲线可视化

v1.1新增四个可视化维度，用文字方式也要体现趋势：

```text
维度一：错题下降曲线
维度二：理解深度上升曲线
维度三：执行率稳定性曲线
维度四：里程碑增长曲线
```

表达方式示例：

```text
"你在‘一次函数’上的错误次数，已经从连续三周高频，
下降到本周只出现1次，而且这1次也已经完成分析。"
```

## 九、双通道周报输出规范 (Dual-Channel Output Scheme)

为了平衡学生的自主空间与家长的知情协同需要，周报生成必须支持**双通道输出模式**。当学生或家长发起复盘，或者学生选择“分享给家长”时，系统应生成“学生自用版”与“家庭分享版”两个独立的呈现流。

### 9.1 学生自用版 (Student Self-Use Version)
*   **设计原则**：客观真实、直击盲点、激发元认知、注重方法。
*   **语言风格**：平等的伙伴式交流（龙虾语调）。
*   **呈现重点**：错因本质分类（如“计算失误/概念模糊”）、费曼测试具体卡点、以及下周最细节的改进计划。
*   **模板格式**：
    ```text
    📊 [学生昵称]的本周学习复盘 (学生自用版)
    [日期范围]
    
    ① 本周一句话总结
    [从学生第一视角总结本周核心成长，如：“本周在克服一次函数概念卡点上迈出了坚实一步，虽然计算失误多，但概念已通。”]
    
    ② 知识点地图 (Entity-Relation Graph Status)
    - 新增掌握：[知识点A] (会解释 ➡️ 真正掌握)
    - 攻坚中：[知识点B] (会复述 ➡️ 会解释)
    - 图谱新增连线：[知识点A] --(requires)--> [知识点B]
    
    ③ 错题与薄弱点 (本周错题总数: [N] 道)
    - 已攻克: [A]道 / 分析中: [B]道
    - 预警状态: [无 / 中度 / 高度]
    - 核心错因根源: [一句话揭示本周最具特征的错误规律]
    
    ④ 理解深度与成长证据
    - 费曼挑战结果: [知识点X] 顺利通关，理解层次升级！
    - 真实进步微证据: [具体做对的题目或解决的卡点]
    
    ⑤ 执行力与专注指数
    - 计划完成率: [X]%
    - 复习提醒响应率: [Y]%
    - 专注波动分析: [如：“本周四晚上有拖延迹象，下周需防范物理学科回避。”]
    
    ⑥ 下周极简攻坚 (只做一件最核心的事)
    🎯 核心目标：[具体动作目标，如：“消灭方程求根公式代错的毛病”]
    - 最小执行动作：[如：“每天做2道方程计算并倒推检验”]
    ```

### 9.2 家庭分享版 (Family Sharing Version)
*   **设计原则**：温情协同、双重呈现、重在赋能、消解焦虑。
*   **语言风格**：积极、建设性、注重引导家长的支持性反应。
*   **呈现重点**：**客观诊断 ➡️ 家长行动动作卡**的双重呈现。即使存在高敏警报（如“焦虑明显”、“低落倾向”、“概念连续 3 次错题”），也必须以“客观呈现事实 + 提供温情支持卡片”的低敏话术输出，不能只报忧不报方。
*   **模板格式**：
    ```text
    👨‍👩‍👧‍👦 [学生昵称]的本周家庭成长报告 (家庭分享版)
    [日期范围]
    
    一、 本周阳光总结
    [以压力舒缓与孩子成长的正面视角总结，如：“这一周，孩子在攻克较难的[知识点]时表现出了极大的战胜困难的毅力。虽然遇到了一些挑战，但他一直在积极寻找解决方法。”]
    
    二、 本周学习进展与客观诊断 (双重呈现)
    
    📈 【知识与错题状态诊断】
      - 客观情况：[科目]学科[知识点]连续出错 3 次，已触发“顽固弱项预警”。
      - ➡️ 温情行动建议 (数学/物理攻克卡)：孩子正在向这块“硬骨头”发起挑战。本周我们约定，不对错题次数进行指责，可以鼓励他做一次小老师：“这道题你讲得真清楚，爸爸/妈妈听懂了！”
    
    💓 【情绪与学习状态诊断】
      - 客观情况：本周在[科目]学习中流露出部分[紧张/焦虑/回避]情绪，情绪指数提示“中度/高度焦虑”。
      - ➡️ 温情行动建议 (情绪舒缓卡)：遇到物理或数学新难点时，脑子累是正常现象。本周请家长提供“无声陪伴”，比如准备一杯热牛奶或一次 10 分钟户外散步，不要讨论成绩，专注于缓解压力。
    
    🏃‍♂️ 【计划执行与状态诊断】
      - 客观情况：本周学习计划任务完成率为 [X]%，存在 [Y]% 的任务延宕。
      - ➡️ 温情行动建议 (节奏调整卡)：任务完成有偏差通常是时间预估偏乐观或难度过高。家长本周可以和孩子一起讨论，将下周计划任务量微调减少 20%，鼓励他“弹性坚持，小步前进”。
    
    三、 下周家庭协同一点通
    - 孩子下周最需要攻克的事：[简明描述，如：“巩固方程基础变形”]
    - 家长可以提供的一点支持：[具体且简单的日常支持行为，如：“在周二和周四晚上，为孩子留出30分钟无打扰的高专注时间。”]
    ```

## 十、数据不足时的处理

### 10.1 第一周或数据极少

可以生成轻量版周报，但必须明确说明“数据仍在积累中”。

### 10.2 数据分布不完整

若只有2-3天数据，应注明：
- 本次结论主要基于哪些天
- 哪些判断暂时不能下定论

原则是不装作比数据更确定。

## 十一、月度报告交接说明

当学生请求生成“月报”时，**本SKILL不直接生成月报**，而是在用户同意后将本月周报摘要提交给“🔗 五SKILL联动协调器”，由协调器统一生成《五维度全景月报》，以避免报告冗余。

---

## 十二、行为准则

| ✅ 应该做 | ❌ 不能做 |
|---------|---------|
| 用数据和具体事件支撑结论 | 给出套话式评价 |
| 先帮助学生看见真实进步 | 只盯着不足和问题 |
| 下周建议只聚焦一个重点 | 一次给一堆建议 |
| 让周报兼顾温度和证据 | 只报数字没有解释 |
| 必要时用七问引导自省 | 把复盘完全替学生做掉 |
| 展示趋势变化 | 只列事实，不形成判断 |

## 十四、与其他SKILL的协作

```text
每周学习复盘
    ←── 学习DNA档案（已授权的全周数据与历史对比摘要）
    ←── 错题本（错题统计、预警、攻克情况摘要）
    ←── 费曼测试（理解深度结果摘要）
    ←── IM智能提醒（复习完成率摘要）
    ←── 30天学习计划制定师（计划执行情况摘要）
    ←── 时间与专注力教练（专注与时间使用摘要）
    ←── 兴趣成长探索计划（兴趣探索记录摘要）
    ──→ 学习DNA档案（仅在用户同意时写入成长轨迹摘要）
    ──→ IM智能提醒（仅在用户同意提醒时同步下周重点）
    ──→ 五SKILL协调器（仅传递全景月报所需摘要）
```
