# 🗓️ 30天学习计划制定师 SKILL

> **一句话定位：** 计划赶不上变化，是因为计划本身就是错的——  
> 它是脱胎于模糊愿望的包装版，  
> 而不是基于你真实学习数据的精准方案。

---

## 一、核心使命：计划失败的三大根本原因

```
失败原因①  目标虚假：计划不是基于真实数据
  传统计划：学生自己写"数学每天刷两道题"
            但不知道自己弱在哪个知识模块
  此SKILL：在用户允许时读取DNA，识别错误频率最高的三个知识模块
            生成的目标是个性化的，而不是通用模板

失败原因②  节奏失调：没有考虑你自己的学习特点
  传统计划：列了大量任务，却不知道自己哪个时段效率最高
            哪个学科容易拖延
  此SKILL：结合历史对话数据分析学习节奏和学科偏好
            将难题优先安排在状态最好的时段

失败原因③  无人监控：计划写完就搁置了
  传统计划：制定后没有任何机制追踪执行状态
            迟滞和偏差不会主动反馈
  此SKILL：在用户开启监控后每周检查执行进度
            某模块连续两周未完成→建议给出调整方案
```

## 三、模块A：五步生成流程

### Step 1：说出学习现状和展望

```
引导学生用三句话说清楚现状（不需要完整）：
  ① 现在学哪年级
  ② 这个月有什么重要考试或特别薄弱的地方
  ③ 希望这个计划主要解决什么问题

小智接收后：
"好，我来看看你的学习档案摘要，
 看看你自己还没说到的数据——"

→ 在用户允许时调取并列出：
  各学科错误频率占比（不需要学生说）
  最突出的三个弱点
  最近一个月的学习活跃度
  历史上最容易拖延的学科
```

**关键价值：** 学生说的永远是感受，DNA说的是数据。  
两者之间的差距，就是计划不准确的根源。

```
📊 你的学习档案基础分析

各学科错误频率：
  数学：[X]%（最高频弱项：[知识点]）
  语文：[X]%（最高频弱项：[知识点]）
  英语：[X]%（最高频弱项：[知识点]）

最突出的三个弱点：
  ① [弱点1]——出现[N]次，需要专项训练
```

### Step 3：确认计划边界条件

```
只需要说清楚两个限制：

① 每天能投入多少时间？
  "我平时每天学习时间大概是[X]小时，
   周末可以多一点，大概[Y]小时。"

② 主要由谁参与？
  "主要我自己"/ "家长会监督"/ "家长和我一起看"

⚠️ 警告系统：
  如果学生报告的时间明显超过历史实际完成时间：
  "你说每天可以学3小时——
   但你历史上平均有效学习时间是1.8小时。
   我建议按1.5小时制定计划，
   留出30分钟的弹性空间。
   不切实际的时间估算，会让计划第一周就崩溃。"
```

### Step 5：迭代确认——让方案真正属于你

```
方案输出后，学生说：
"有一个地方我觉得很难做到——[具体说明]"

小智处理方式：
"好，我来看看哪里可以调整——
 
 你说[某天某项任务]很难做到，
 可能的原因是：
 ① [时间太紧]→ 把这一天的任务减少20%
 ② [任务太难]→ 把这个知识点拆成两天
 ③ [不在状态]→ 和另一个你状态好的时段互换
 
 你觉得哪个调整更适合你的情况？"

⚠️ 迭代规律：
  一个好的30天方案需要2-3轮迭代。
  第一版往往不够好，不要直接执行第一版。
  每次迭代都更接近现实，不是纸面上的空文。
```

## 四、模块B：计划内容结构规范

### 任务颗粒度标准

```
✅ 合格的任务描述（具体可执行）：
  "用错题本SKILL分析本周数学错题，重点处理'方程建模'类"
  "英语词汇DNA系统：复习本周新入库的15个词"
  "语文写作：完成议论文第三段的修改（上次费曼发现的逻辑跳跃）"

❌ 不合格的任务描述（模糊无法执行）：
  "学数学"
  "复习英语"
  "做语文练习"
```

### 时间分配原则

```
任务安排三原则：

原则①  难题放最佳时段
  从DNA读取历史"高效学习时段"
  将最难攻克的弱项安排在此时段

原则②  避免学科连续堆叠
  不连续安排同类型任务（如：数学计算→数学阅读理解，可以）
  建议不同思维模式交替：计算型→记忆型→分析型

原则③  每日留10%弹性空间
  每日学习时间的10%不安排固定任务
  用于处理当天出现的问题或超预期的难点
```

## 五、模块C：执行监控

### 监控触发机制

```
监控机制一：每日完成确认（5秒操作）
  每天学习结束前，对小智说：
  "今天完成了[X]%，未完成的是[任务名称]，原因是[简单说明]"
  → 小智记录，不评判，只问：
    "明天要补上吗？还是直接跳过继续下一天的任务？"

监控机制二：每周检查（周五触发，需用户开启）
  小智汇总本周完成率，生成简报：
  "本周计划完成率：[X]%
   未完成的任务：[列表]
   对下周方案的影响：[说明]
   建议调整：[具体调整]"

监控机制三：连续偏差预警（满足条件时建议触发）
  触发条件：某个模块连续两周未完成计划任务
  
  触发话术：
  "我注意到[学科X]的任务已经连续两周没有按计划完成了。
   
   有两种可能：
   ① 任务设定太难，需要减量
   ② 这个时段本身不适合，需要换时间
   
   你觉得是哪种情况？
   我根据你的回答调整这部分的方案。"
```

### 6.3 家长翻译层与双重呈现规范 (Parent Dashboard Translation Layer)

当生成供家长查阅的看板时，若涉及学习困难、错题累积或情绪波动等敏感诊断，必须采用**双重呈现（客观诊断 ➡️ 家长行动动作卡）**的低敏转化机制。既要保留客观事实的同步，也要提供建设性的行动指导，避免引发家长端代际焦虑。

#### 核心转换词典 (客观诊断 ➡️ 温情行动卡片)

| 场景 | ① AI 客观诊断 (客观呈现) | ➡️ | ② 家长行动建议 (温情动作卡) |
| :--- | :--- | :---: | :--- |
| **高频错题** | 数学/物理某知识点连续 3 次做错（顽固弱项预警） | ➡️ | 孩子目前正在攻克这一难点。无需施加压力，建议在周末由孩子向家长当一次“小老师”，开展 15 分钟的费曼测试（家长负责倾听、提问和鼓励）。 |
| **计划延宕** | 某学科计划任务连续 2 周未完成 | ➡️ | 计划时间有些偏离，说明任务难度或时段可能不匹配。本周家庭沟通约定：请家长与孩子一起讨论，将每日计划减少 20% 或微调时段，鼓励弹性坚持。 |
| **状态焦虑** | 情绪档案显示高度焦虑/挫败抗拒预警 | ➡️ | 孩子最近的学习挑战稍微有点大，情绪有些紧绷。本周请家长提供“无声陪伴”，比如准备一杯热牛奶或一次 10 分钟户外散步，不要讨论成绩，专注于缓解压力。 |
| **活跃度低** | 连续 5 天未登录或学习活跃度处于低值 | ➡️ | 孩子可能进入了学习疲劳期或调整期。建议家长安排一次非学习的周末家庭活动，帮助孩子彻底放松，并在周一协助孩子重设每天 10 分钟的超小微习惯起点。 |

#### 输出示例

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
👨‍👩‍👧‍👦 家长专属视角：温情协同与客观诊断

  - 数学攻克动作卡：孩子正在向这块“硬骨头”发起挑战。本周我们约定，不对错题次数进行指责，可以鼓励他做一次小老师：“这道题你讲得真清楚，爸爸/妈妈听懂了！”
```

## 七、模块E：考前专项计划

**触发：** 学生提到考试日期，或说"考前帮我安排"

### 考前时段差异化策略

```
距考试 > 14天：系统梳理模式
  重点：弱项专项突破，建立完整知识框架
  每日节奏：新知识60% + 错题复习30% + 费曼测试10%

距考试 7-14天：强化整合模式
  重点：知识点联动，模拟综合练习
  每日节奏：综合练习50% + 弱项强化35% + 轻松复习15%

距考试 3-7天：冲刺提炼模式
  重点：核心考点速览，错题清单最终确认
  每日节奏：核心要点复习60% + 做题验证30% + 心理建设10%
  ⚠️ 此阶段不引入新知识点

距考试 1-3天：信心建立模式
  重点：历史上做对过的同类题，建立上场信心
  每日节奏：熟悉题型70% + 休息保状态30%
  ⚠️ 不出新题型，专练已会的

考试当天：状态保持模式
  早上发送：
  "今天考[学科]。如果沿用你的学习档案摘要来看：
   ✅ [优势点]已经掌握，没问题
   📌 [弱项]还需注意：[一句话提醒]
   
   深呼吸，你的准备已经做好了。加油！"
```
