# 物理CLAW专项模板

> 本文档为 `xiaozhi-physics-problem-coach` 的配套参考资源，定义5套物理学科专属CLAW模板。
> 每套模板包含：触发短语识别→CLAW结构→适配响应流程。
> 物理CLAW模板与数学CLAW模板的核心区别：物理模板的第一步永远是"画图"。

---

## CLAW框架回顾

```
C - Condition：你遇到了什么情况？
L - Law/Logic：你用了什么物理规律/方法？
A - Attempt：你尝试了什么？走到了哪一步？
W - Want：你想要什么帮助？

物理CLAW核心原则：
  → C必须包含"你画图了吗"
  → L必须是物理规律而非数学公式
  → A必须包含图景建立进度
  → W不能是"给我答案"，而是"帮我分析"或"引导我思考"
```

## 模板① 图景建立类

### 触发短语

```
"我画不出受力分析图"
"我不知道电路图怎么画"
"我不会画光路图"
"这道题我不知道该画什么图"
"我看了题但脑子一片空白"
"我不知道从哪里开始分析"
"这道力学题我找不到受力对象"
```

### CLAW结构

```
C（情况）：
  "我在做[力学/电学/光学]题，
   题目描述了[简要描述物理情境]，
   但我画不出[受力分析图/等效电路图/光路图/运动过程图]。"

L（规律）：
  "我知道应该先画图再分析，
   但我不知道[选哪个物体/判断串并联/确定入射面/分解运动过程]。"

A（尝试）：
  "我[尝试画了但漏了力/试着判断了但不确定/完全不知道从哪开始]。"

W（需求）：
  "帮我分步引导画出这张图，不要直接画给我看，
   而是一步一步让我自己画出来。"
```

### 适配响应流程

```
Step 1：确认图景类型
  AI话术："这道题属于[力学/电学/光学]题，我们需要先画[图景类型]。"

Step 2：分步引导画图（严格按physics-diagram-guide.md）
  受力分析图：
    → "第一步：确定分析对象——这道题分析的是哪个物体？"
    → "第二步：画重力——竖直向下，标G。"
    → "第三步：找接触面——和哪些东西接触？每个接触面可能产生什么力？"
    → "第四步：检验平衡——水平方向和竖直方向力分别平衡了吗？"

  等效电路图：
    → "第一步：从正极出发，电流有几条路径？"
    → "第二步：电压表当断路、电流表当导线，画出简化电路。"
    → "第三步：标出电流方向和各部分电压关系。"

  光路图：
    → "第一步：光线从什么介质进入什么介质？"
    → "第二步：画出法线，标注入射角。"
    → "第三步：根据反射/折射定律画出光线路径。"

Step 3：检查图景完整性
  AI话术："你的图画好了——我们检查一下：
   [重力画了吗？/每个开关状态都画了吗？/法线画了吗？]"

Step 4：进入四步法S2（物理建模）
  图景正确 → "图画对了，接下来我们根据图景选择物理模型。"

Step 5：错误归档（如适用）
  → 图景建立失败 → 归档为P类错误
  → 具体子类型：P01/P02/P03/P04（按图景类型）
```

### 变体场景

```
变体A：多对象图景
  触发："两个物体叠在一起，我不知道分析哪个"
  处理："分别画！先画上面的物体受力图，再画下面的物体受力图。
         用牛顿第三定律把它们联系起来——上面物体对下面的压力
         等于下面物体对上面的支持力。"

变体B：多状态电路
  触发："开关有好几种状态，我不知道怎么处理"
  处理："每种状态画一张等效电路图。
         开关S1闭合S2断开→画一张；S1断开S2闭合→再画一张。
         不能混在一起画。"

变体C：完全空白
  触发："我完全不知道该画什么"
  处理：降级引导→"这道题提到了什么物理现象？[力/运动/电流/光线]
         → 有力→画受力分析图；有运动→画运动过程图；
           有电流→画等效电路图；有光线→画光路图。
         先告诉我题目描述了什么，我帮你判断。"
```

## 模板② 错题分析类

### 触发短语

```
"这道力学题我做错了"
"这道电学题我算的结果不对"
"我物理考试扣了很多分"
"这道题我选了B但答案是C"
"我的计算结果和答案不一样"
"这道浮力题我做错了但不知道哪里错了"
```

### CLAW结构

```
C（情况）：
  "这道[力学/电学/光学]题我做错了，
   我的答案是[学生答案]，正确答案是[正确答案]。"

L（规律）：
  "我用了[物理规律/公式名称]，
   但[结果不对/方向搞反了/过程遗漏了]。"

A（尝试）：
  "我[画了受力图但可能漏了力/列了式子但不知道哪里算错了]，
   我不确定错误出在哪一步。"

W（需求）：
  "帮我找到错误的具体位置和原因，
   不要直接告诉我正确解法，
   让我自己发现哪里错了。"
```

### 适配响应流程

```
Step 1：定位错误步骤（按四步法逐步骤回溯）
  AI话术："我们按步骤来检查——你画图了吗？画完图列的式子还是直接列的？"

  检查顺序（按优先级）：
  ① 图景是否正确？→ 不画图就列式 → P类错误（最高优先级）
  ② 物理模型是否正确？→ 模型选错 → C类错误
  ③ 公式是否正确？→ 公式用错/条件不满足 → F类错误
  ④ 计算过程是否正确？→ 数学执行错误 → T类错误

Step 2：逐步还原学生的解题过程
  AI话术："把你做的每一步说给我听——
   第一步你画了什么图？第二步你选了什么模型？
   第三步你列了什么公式？第四步你怎么算的？"

Step 3：定位分歧点
  AI话术："你的过程到第[N]步之前都是对的，
   第[N]步开始出了问题——这一步你是怎么想的？"

Step 4：根因追问（不用三层次追问，用简化版）
  "你第[N]步漏了[摩擦力/某个过程/某个开关状态]——
   为什么会漏？是因为没画图，还是画了图没看仔细？"
  → 没画图 → P类根因
  → 画了图但看漏 → P类（图景意识不足）
  → 画了图但概念理解错 → C类根因

Step 5：针对性修复
  → P类：重画图景，建立图景意识
  → C类：概念澄清（联动概念直觉器）
  → F类：公式适用条件检查
  → T类：数学计算自检

Step 6：出同类题验证
  "我出一道类似的题，你用正确的方法做一遍——
   注意刚才出错的那一步。"

Step 7：归档到物理错误DNA
  → 记录：错误维度、子类型、根因、修复方法
```

## 模板③ 建模卡壳类

### 触发短语

```
"我知道用F=ma但不知道怎么列方程"
"我知道是欧姆定律但不知道怎么设未知数"
"我画了受力图但不知道怎么算"
"我知道是串联分压但不知道从哪入手"
"公式都认识但不知道怎么组合"
"图景画出来了但不知道下一步做什么"
```

### CLAW结构

```
C（情况）：
  "我在做[力学/电学]题，
   我已经画了[受力图/等效电路图]，
   但我不知道怎么从图景转化成方程。"

L（规律）：
  "我知道要用[物理规律名称]，
   但我不知道[怎么设未知数/怎么组合公式/从哪个条件入手]。"

A（尝试）：
  "我[列了一个式子但不够/试了但不知道对不对/完全不知道从哪开始列]。"

W（需求）：
  "帮我从图景过渡到方程，
   引导我自己列出正确的方程组，
   不要直接给我完整解法。"
```

### 适配响应流程

```
Step 1：确认图景正确性
  AI话术："先确认你的图景画对了——[展示图景检查点]。
   图景对了才能列式，图景错了列出来的式子肯定不对。"

Step 2：从图景中提取物理关系
  AI话术："看看你的图——
   受力图：哪些力平衡？哪些力不平衡？→ 平衡方程 or F=ma
   电路图：串并联关系？各部分电压电流关系？→ 欧姆定律+串并联规律
   运动图：每段运动遵循什么规律？→ 运动学公式"

Step 3：引导设未知数
  AI话术："要求什么？——把它设为未知数。
   还需要什么中间量？——也设出来。
   你有几个未知数？需要几个方程？"

Step 4：逐个列方程
  AI话术："从图景中，你能找到几个独立的物理关系？
   每个关系写一个方程。
   比如：竖直方向平衡→N+Tcosθ=G
         水平方向平衡→Tsinθ=f
   你自己找找看。"

Step 5：检查方程数量
  AI话术："你有[N]个未知数，[M]个方程——够吗？
   不够的话，图景里还有没用的信息。回去看看图。"

Step 6：引导求解（不直接算）
  AI话术："方程列好了——你自己解解看。
   解到哪一步卡住了告诉我。"

Step 7：如果反复卡壳
  → 联动物理建模教练SKILL
  → 归档为C类或F类错误（概念理解不足或公式应用困难）
```

## 模板④ 实验分析类

### 触发短语

```
"这个实验题我不会分析"
"我不知道怎么设计实验验证结论"
"控制变量法我不太会用"
"实验数据的规律我找不出来"
"这个实验的误差分析怎么做"
"我不知道怎么从实验数据得出结论"
```

### CLAW结构

```
C（情况）：
  "我在做物理实验题，
   实验内容是[实验名称/简要描述]，
   我不知道怎么[设计实验/分析数据/判断结论/处理误差]。"

L（规律）：
  "我知道要用[控制变量法/转换法/类比法等]，
   但我不知道[控制什么变量/怎么转换/怎么判断]。"

A（尝试）：
  "我[试着分析了但不确定对不对/看了数据但找不出规律/完全不知道从哪开始]。"

W（需求）：
  "帮我理清实验分析思路，
   引导我自己找出实验结论，
   不要直接告诉我答案。"
```

### 适配响应流程

```
Step 1：明确实验目的
  AI话术："这个实验要验证什么/探究什么？
   知道目的，才能判断方法对不对。"

Step 2：识别实验方法
  AI话术："这个实验用了什么方法？——
   控制变量法：探究A对B的影响，控制其他量不变
   转换法：把不容易测的量转换成容易测的量
   等效替代法：用一个效果相同的量替代另一个
   你判断是哪种？"

Step 3：分析实验步骤（针对控制变量法）
  AI话术："控制变量法的核心——
   要探究A对B的影响：
   ① 控制其他变量不变
   ② 只改变A
   ③ 观察B怎么变
   你能说出这个实验控制了什么、改变了什么、观察了什么吗？"

Step 4：数据处理引导
  AI话术："看数据——
   同一列数据有什么规律？
   不同列数据之间有什么关系？
   能用公式表示吗？"

Step 5：误差分析引导
  AI话术："实验结果和理论值有差距——
   可能的误差来源：
   ① 测量误差（读数不准、仪器精度）
   ② 系统误差（方法本身的偏差）
   ③ 操作误差（操作不规范）
   你觉得这个实验的主要误差来自哪里？"

Step 6：联动物理实验思维教练
  → 实验题深度分析应联动实验思维教练SKILL
  → 本模板只做初步引导，不替代实验思维训练
```

## 模板⑤ 考前突击类

### 触发短语

```
"明天考物理帮我复习"
"后天考试帮我梳理电学"
"物理考试前我想快速过一遍"
"帮我列出力学必考知识点"
"考前帮我检查还有什么弱项"
"明天考试但我电学还不会"
```

### CLAW结构

```
C（情况）：
  "[明天/后天/N天后]要考物理，
   考试范围是[力学/电学/全部]，
   我[对哪些内容没把握/想知道重点是什么]。"

L（规律）：
  "我学过这些内容但[记不清公式/不确定方法/容易在[某类题]上出错]。"

A（尝试）：
  "我[复习了一些但不够系统/做了几道题但不确定对不对/还没开始复习]。"

W（需求）：
  "帮我快速梳理[知识模块]的核心考点和解题方法，
   重点标注我的弱项和易错点，
   不要从头讲一遍，要有针对性地复习。"
```

### 适配响应流程

```
Step 1：确定考试范围和时间
  AI话术："考试范围是什么？还有多少时间？
   → 明天考：只做核心梳理+高频错题
   → 3天后考：核心梳理+弱项专练
   → 一周后考：完整复习+系统训练"

Step 2：知识框架快速梳理（不从头讲）
  力学框架：
    "力学核心三条线：
     ① 力线：重力→弹力→摩擦力→力的合成与平衡
     ② 运动线：匀速→变速→v-t图→运动过程分析
     ③ 应用线：压强→浮力→简单机械→功和功率
     你哪条线最弱？"

  电学框架：
    "电学核心三条线：
     ① 电路线：串并联→等效电路→动态电路
     ② 规律线：欧姆定律→电功率→焦耳定律
     ③ 实验线：伏安法→测电功率→电磁感应
     你哪条线最弱？"

Step 3：图景绘制自查表
  "考试前检查——
   □ 力学题：看到力就先画受力分析图
   □ 电路题：看到开关就画等效电路图
   □ 运动题：看到过程就画运动过程图
   □ 光学题：看到光线就画光路图
   不画图就列公式 → 必扣分"

Step 4：公式适用条件速查
  "考试常考的公式适用条件——
   □ 欧姆定律：只适用于纯电阻电路
   □ 密度公式：只适用于均匀物质
   □ 浮力公式（阿基米德）：适用于液体和气体
   □ 功的公式W=Fs：F和s必须同方向
   □ 杠杆平衡条件：力臂是支点到力的作用线的距离"

Step 5：单位统一检查
  "考试最容易扣分的地方——单位不统一：
   □ km/h → m/s（÷3.6）
   □ cm² → m²（÷10000，不是÷100！）
   □ g → kg（÷1000）
   □ kW·h → J（×3.6×10⁶）
   □ mm → m（÷1000）"

Step 6：错误档案对照
  "从你的物理错误DNA看——
   近期最频繁的错误是[维度+子类型]。
   考试时特别注意[具体提醒]。"
  → 调取 xiaozhi-physics-error-dna 的近期弱项清单
  → 重点标注"顽固弱项"

Step 7：3道高频题型快速练
  "做3道高频考题——每道只做2分钟：
   做对了→这个点OK
   做错了→标记，考试遇到先画图再分析"
  → 不进入完整四步法流程
  → 快速判断+即时反馈
  → 错误记录但不归档（考前模式，不增加焦虑）

考前模式特殊规则：
  → 不启动苏格拉底三层次追问（时间不够）
  → 不归档到错误基因档案（避免考前增加压力）
  → 重点放在"应试技巧"而非"深层理解"
  → 鼓励画图但不强制完整四步法
```

| 模板 | 触发信号 | 核心难点 | 图景要求 | 追问深度 | 归档优先级 |
|------|---------|---------|---------|---------|-----------|

## 物理CLAW与数学CLAW的核心区别

```
数学CLAW第一步：让学生先自己算
物理CLAW第一步：让学生先画图

数学CLAW关注：思路→方法→计算→验证
物理CLAW关注：图景→模型→公式→检验

数学CLAW的"规律"是数学定理和公式
物理CLAW的"规律"是物理定律和适用条件

数学CLAW的"尝试"是计算过程
物理CLAW的"尝试"必须包含图景建立进度

数学CLAW的"需求"通常是"帮我找到思路"
物理CLAW的"需求"通常是"帮我看清物理现象"或"帮我建立图景"

关键区别：
  数学：读懂题 → 找方法 → 答案
  物理：读懂题 → 画图景 → 找规律 → 列式 → 检验
                ↑
          这一步是物理独有的，也是学生最容易跳过的
          物理CLAW模板必须确保这一步不被跳过
```
