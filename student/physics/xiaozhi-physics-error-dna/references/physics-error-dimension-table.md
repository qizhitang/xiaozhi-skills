# 物理错因维度表

> 本文档为 `xiaozhi-physics-error-dna` 的核心参考资源，定义物理学科包的五维错因维度体系、子类型分类、跨维度关联规则与追踪标准。

---

## 一、维度总览

```
物理错因维度体系（五维×多子类型）

维度①  图景建立错误 (P类) ── 物理独有的认知层错误
  "看不见"物理世界——不会画图、不会建立物理图景

维度②  概念混淆 (C类) ── 理解层错误
  物理概念理解偏差或相似概念混用

维度③  公式误用 (F类) ── 应用层错误
  知道公式但用错了——条件/单位/方向

维度④  过程分析错误 (R类) ── 分析层错误
  物理过程分析不完整或阶段划分错误

维度⑤  数学工具错误 (T类) ── 工具层错误
  物理思路对但数学执行出错

每道错题必须归入且只归入一个维度（主维度）。
如存在跨维度因素，记录为"主维度+次要维度"。

优先处理顺序：P类(图景) > C类(概念) > R类(过程) > F类(公式) > T类(数学)
→ 图景是一切物理分析的起点，图景错误会导致后续全错
```

## 二、维度①：图景建立错误

**定义：** 不会将文字描述转化为物理图景（受力分析图、运动过程图、电路图、光路图），导致后续分析缺乏可视化基础。

**判断标准：** 学生拿到题后直接列公式，不画图，或画的图有明显遗漏/错误。指出图景问题后能正确分析 → P类为主。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发学段 | 关联参考 |
|---------|------|---------|---------|---------|
| P01 | 不画受力分析图 | 凭感觉列方程，遗漏某个力 | 八年级力学 | → C01力概念混淆 |
| P02 | 运动过程分析不清 | 不会画v-t图/s-t图；过程转折点遗漏 | 八年级力学 | → R01过程阶段遗漏 |
| P03 | 电路图识别错误 | 串并联判断错误；等效电路画错 | 九年级电学 | → R03电路状态分析错误 |
| P04 | 光路图画错 | 反射角/折射角方向搞反；透镜成像位置错 | 八年级光学 | → C05光学概念混淆 |
| P05 | 参考系选取错误 | 以运动物体为参考系导致方向混乱 | 八年级力学 | → C02运动学概念混淆 |
| P06 | 隐含条件未图景化 | "光滑""轻质""缓慢"等条件没体现在图中 | 八-九年级 | → F01公式适用条件忽略 |
| P07 | 多对象关系图缺失 | 两个物体/两个电路状态不会分别画图再找关系 | 九年级 | → R02多过程关联遗漏 |

**严重程度分级：**

```
轻度（偶发）：P05 参考系选取错误
  → 处理：提醒即可，不启动专项训练

中度（模式化）：P01、P03、P04、P06
  → 处理：专项图景训练 + "先画图再列式"强制流程
  → 目标：建立"读题→画图→分析"的条件反射

重度（顽固）：P02、P07
  → 处理：图景重建训练 + 过程分析训练（联动建模教练和实验思维教练）
  → 运动过程图和多对象图景是最难建立的物理直觉
```

**根治方法映射：**

```
P01 不画受力图 → 受力分析四步法训练（选对象→画重力→画接触力→检验平衡）
  → 辅助工具：physics-diagram-guide.md 受力分析追问序列

P02 运动过程不清 → 运动过程图三步法（标状态→画过程→标转折）
  → 辅助工具：physics-diagram-guide.md 运动过程图追问序列

P03 电路图识别错误 → 电路分析三步法（识别串并联→画等效电路→标电流方向）
  → 辅助工具：physics-diagram-guide.md 电路图追问序列

P04 光路图画错 → 光路作图法（确定介质→确定入射面→画反射/折射路径）
  → 辅助工具：physics-diagram-guide.md 光路图追问序列

P06 隐含条件未图景化 → 关键词→图景标记训练
  → 建立"看到光滑→图中标注无摩擦""看到缓慢→图中标注近似平衡"的条件反射
```

## 三、维度②：概念混淆

**定义：** 对物理概念的理解有偏差，或把两个相似但本质不同的物理概念混用。

**判断标准：** 学生说"我知道这个概念但分不清它和那个的区别"，或对概念含义描述含糊。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发学段 | 关联参考 |
|---------|------|---------|---------|---------|
| C01 | 力的概念混淆 | 平衡力vs作用力反作用力；质量vs重力 | 八年级 | → concept-confusion-map §力学 |
| C02 | 运动学概念混淆 | 速度vs速率；路程（初中）——⚠加速度、位移属高中，初中不作要求 | 八年级/高中 | → concept-confusion-map §力学 |
| C03 | 压强概念混淆 | 压强vs压力；固体压强vs液体压强计算方法混用 | 八年级 | → concept-confusion-map §力学 |
| C04 | 浮力概念混淆 | 浮力方向判断错；浮力与排开液体重量的关系理解偏差 | 八年级 | → concept-confusion-map §力学 |
| C05 | 光学概念混淆 | 反射vs折射混用；实像vs虚像判断错 | 八年级 | → concept-confusion-map §光学 |
| C06 | 电学概念混淆 | 电流vs电压；电功vs电功率；短路vs断路 | 九年级 | → concept-confusion-map §电学 |
| C07 | 能量概念混淆 | 功vs能；动能vs势能；机械效率vs功率 | 八-九年级 | → concept-confusion-map §力学/电学 |
| C08 | 热学概念混淆 | 温度vs热量；蒸发vs沸腾；比热容vs热值 | 八年级 | → concept-confusion-map §热学 |

**严重程度分级：**

```
轻度：单一概念理解偏差 → 可通过生活类比一次解决
  → 处理：联动物理概念直觉器，一次类比追问重建理解

中度：两个概念混淆 → C01/C03/C06类型（最容易混淆的概念对）
  → 处理：概念对照表 + 追问 + 3道对比题
  → 工具：concept-confusion-map.md 对照追问

重度：根基概念缺失 → 影响多个后续概念
  → 如：力的概念不清 → 受力分析、力的合成、压强、浮力全部受影响
  → 处理：溯源补充前置概念，重建概念链
```

**根治方法映射：**

```
C类错误通用根治路径：
  ① 追问定位：用三层次追问第一层（现象层）确认混淆方向
  ② 类比重建：用物理概念直觉器SKILL提供生活类比
  ③ 对照区分：用concept-confusion-map做相似概念对比
  ④ 应用验证：出3道纯概念应用题确认理解
  ⑤ 迁移确认：出1道变形题确认不是只会原题

关键区分原则：
  → 如果学生说"我懂了"但同类题还是错 → C类（概念混淆）
    而不是T类（数学工具错误）
  → 区分标准：给一个简化版图景，概念判断做对了吗？
    做对了 → T类（计算执行错误）
    做错了 → C类（概念本身理解偏差）
```

## 四、维度③：公式误用

**定义：** 知道公式但用错了——适用条件不满足、单位不统一、方向不考虑等。

**判断标准：** 学生能说出公式，但代入的条件、单位或方向有误。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发学段 | 关联参考 |
|---------|------|---------|---------|---------|
| F01 | 公式适用条件忽略 | 密度公式用于混合物；欧姆定律用于非纯电阻 | 八-九年级 | → P06隐含条件未图景化 |
| F02 | 物理量单位不统一 | km/h与m/s混用；kW与W混用 | 八-九年级 | → T01单位换算错误 |
| F03 | 矢量方向未考虑 | 力的合成用代数加法；速度方向忽略 | 八年级 | → C01力的概念混淆 |
| F04 | 公式变形错误 | 从ρ=m/V推导m=ρV时搞反；I=U/R变形出错 | 八-九年级 | → T03代数运算错误 |
| F05 | 常数代入错误 | g=9.8取10时忘记说明；水的密度值记错 | 八-九年级 | — |
| F06 | 多公式联立错误 | 串联电路多步骤推理时公式选择混乱 | 九年级 | → R03电路状态分析错误 |
| F07 | 图像信息读取公式对应错误 | v-t图面积含义搞错；I-U图斜率含义搞错 | 八-九年级 | → P02运动过程分析不清 |

**严重程度分级：**

```
轻度：F05 常数记错 → 提醒即可
  → 处理：提供常数速查表

中度：F01、F02、F03、F04 → 系统性公式应用偏差
  → 处理：公式适用条件卡片 + 单位统一自检清单
  → 工具：physics-math-tools-checklist.md 自检步骤

重度：F06、F07 → 多公式联立或图像-公式对应失败
  → 处理：建模训练（联动物理建模教练）
  → 目标：建立"物理图景→选择公式→代入数据→检验单位"的完整流程
```

**根治方法映射：**

```
F01 公式适用条件忽略 → 公式卡片法
  → 每个公式写明：名称 / 表达式 / 适用条件 / 典型误用场景
  → 建立"用公式前先检查条件"的习惯

F02 单位不统一 → 单位统一自检法
  → 解题第一步：统一为国际单位制
  → 工具：physics-math-tools-checklist.md 单位换算表

F03 矢量方向 → 图景标注法
  → 在图景中用箭头标注所有矢量方向
  → 合成时用平行四边形法则，不是代数相加

F06 多公式联立 → 建模三步法
  → 联动物理建模教练：识别现象→选择模型→数学表达
```

## 五、维度④：过程分析错误

**定义：** 物理过程分析不完整，遗漏了关键阶段，或多个过程之间的关联关系理解错误。

**判断标准：** 学生能画出基本图景，但过程阶段有遗漏，或前后过程的衔接点搞错。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发学段 | 关联参考 |
|---------|------|---------|---------|---------|
| R01 | 过程阶段遗漏 | 多阶段运动只分析了一段；物态变化遗漏 | 八-九年级 | → P02运动过程分析不清 |
| R02 | 多过程关联遗漏 | 前一阶段的末状态不是下一阶段的初状态 | 九年级 | → P07多对象关系图缺失 |
| R03 | 电路状态分析错误 | 开关闭合/断开前后电路结构变化没分析 | 九年级 | → P03电路图识别错误 |
| R04 | 能量转化路径遗漏 | 摩擦生热没算；电能→热能+光能拆分错误 | 八-九年级 | → C07能量概念混淆 |
| R05 | 极值/临界条件遗漏 | "刚好离开""最大距离"等临界条件没分析 | 八-九年级 | → F01公式适用条件忽略 |
| R06 | 实验过程设计缺陷 | 控制变量遗漏；测量顺序错误 | 八-九年级 | → 实验思维教练联动 |

**严重程度分级：**

```
轻度：R01 单一过程遗漏 → 补充即可
  → 处理：提示"还有其他过程吗？"

中度：R02、R03、R04 → 多过程/多状态关联遗漏
  → 处理：过程图训练 + "每个转折点单独画图"习惯
  → 特别：R03电路状态分析是九年级物理最大失分点之一

重度：R05、R06 → 临界条件分析缺失或实验设计缺陷
  → 处理：临界条件专项训练（联动物理建模教练）
  → 实验设计训练（联动物理实验思维教练）
```

**根治方法映射：**

```
R类错误通用根治路径：
  ① 过程拆分：把完整物理过程拆成若干阶段
  ② 逐段画图：每个阶段画一张独立的图景
  ③ 找衔接点：前一段的末状态 = 下一段的初状态
  ④ 分别列式：每段独立列方程
  ⑤ 联立求解：用衔接点关系联立方程

R03 电路状态分析专项：
  → 每个开关状态画一张等效电路图
  → 标注：开关闭合/断开 → 串并联结构变化 → 总电阻变化 → 总电流变化
  → 工具：physics-diagram-guide.md 电路图三步法
```

## 六、维度⑤：数学工具错误

**定义：** 物理分析思路正确，但在数学执行层面出错——单位换算、比例运算、代数计算、几何关系等。

**判断标准：** 学生物理概念清楚、图景正确、公式选择对，但最终计算结果不对。

### 子类型分类

| 子类型ID | 名称 | 典型表现 | 高发学段 | 关联参考 |
|---------|------|---------|---------|---------|
| T01 | 单位换算错误 | cm²→m²除以100而非10000；kW·h→J换算错误 | 八-九年级 | → F02物理量单位不统一 |
| T02 | 比例运算错误 | 串联分压/并联分流比例计算出错 | 九年级 | → F06多公式联立错误 |
| T03 | 代数运算错误 | 解方程出错；代入数值计算出错 | 八-九年级 | → F04公式变形错误 |
| T04 | 几何关系错误 | 杠杆力臂找错；光路几何角度算错 | 八年级 | → P04光路图画错 |
| T05 | 图像读取错误 | 坐标轴含义搞错；比例尺忽略 | 八-九年级 | → F07图像信息读取错误 |
| T06 | 科学计数法错误 | 10⁻³写成10³；指数运算出错 | 八-九年级 | — |
| T07 | 近似与精度处理错误 | 中间步骤取整导致最终结果偏差大 | 八-九年级 | — |

**严重程度分级：**

```
轻度：T05、T06、T07 → 偶发性数学操作错误
  → 处理：提醒注意，不启动专项训练

中度：T01、T02、T03 → 系统性数学工具薄弱
  → 处理：数学工具自检清单 + 专项练习
  → 工具：physics-math-tools-checklist.md
  → 关键：区分是"物理问题"还是"数学问题"

重度：T04 几何关系错误 → 可能是图景问题而非数学问题
  → 处理：先确认图景是否正确，再处理几何计算
  → 如力臂找错 → 可能是P01（不会画受力图）而非T04
```

**根治方法映射：**

```
T类错误核心原则：
  → 先确认物理分析没错（图景对、概念对、公式对）
  → 再针对数学薄弱点做专项训练
  → 不在物理SKILL中做纯数学教学，而是建立"物理情境→数学执行"的桥梁

T01 单位换算 → 国际单位制速查表 + 每次计算前统一单位
T02 比例运算 → 串联分压/并联分流公式推导训练（理解而非记忆）
T03 代数运算 → 分步计算法（每步只做一次运算，减少出错机会）
T04 几何关系 → 图景标注法（在图中直接标注力臂、角度、距离）
```

## 七、跨维度关联规则

```
维度间常见交叉：

P01(不画受力图) + C01(力概念混淆)
  → 判断：画了受力图后能列对式子吗？
    能 → P类为主（图景缺失导致概念无法正确应用）
    不能 → C类为主（概念本身有偏差）

P06(隐含条件未图景化) + F01(公式适用条件忽略)
  → 判断：指出隐含条件后，能用对公式吗？
    能 → P类为主（图景意识不足导致条件遗漏）
    不能 → F类为主（即使条件明确了也不会用公式）

F02(单位不统一) + T01(单位换算错误)
  → 判断：知道需要统一单位但换算出错？
    是 → T类为主（数学工具问题）
    否(不知道要统一) → F类为主（物理公式意识问题）

R01(过程阶段遗漏) + P02(运动过程分析不清)
  → 判断：画了过程图后能完整分析吗？
    能 → P类为主（图景不清导致过程遗漏）
    不能 → R类为主（分析能力本身不足）

R03(电路状态分析错误) + P03(电路图识别错误)
  → 判断：画出等效电路后能正确分析吗？
    能 → P类为主（图景缺失导致分析错误）
    不能 → R类为主（过程分析能力不足）

C06(电学概念混淆) + F06(多公式联立错误)
  → 判断：明确概念后能选对公式吗？
    能 → C类为主（概念混淆导致公式选错）
    不能 → F类为主（公式应用本身有问题）

跨维度记录格式：
  主维度：[维度ID+子类型ID]
  次要维度：[维度ID+子类型ID]（如有）
  交叉判断依据：[图景辅助后结果/条件明确后结果]

优先处理顺序：
  P类(图景) > C类(概念) > R类(过程) > F类(公式) > T类(数学)
  → 图景是物理分析的起点，图景错了一切都错
  → 概念是图景的语言化，概念错了图景无法正确建立
  → 过程分析依赖正确的图景和概念
  → 公式应用建立在正确的分析之上
  → 数学工具是最后一层，解决前面的层次问题后自然改善
```

## 八、顽固弱项追踪标准

```
同一子类型（相同维度+子类型ID+相同知识点标签）出现次数追踪：

出现1次 → 标记"偶发"，低优先级
出现2次 → 标记"初步弱项"，进入观察期
出现≥3次 → 标记"顽固弱项"，启动专项突破流程

顽固弱项突破流程（物理版）：
  Step 1：纯净版图景测试
    给一个简化版物理图景（只含该知识点，排除干扰）
    "先画一下这道题的受力图/电路图：[简化版题目]"
    → 图景画对了 → 问题在干扰条件下的应用 → 出3道递进干扰题
    → 图景画错了 → 根源在图景建立或概念理解 → 联动概念直觉器或图景训练

  Step 2：专项训练（针对子类型的根治方法）
    P类 → 图景绘制专项训练
    C类 → 类比重建 + 对照区分训练
    F类 → 公式适用条件卡片 + 单位自检训练
    R类 → 过程拆分 + 逐段画图训练
    T类 → 数学工具自检清单 + 专项计算训练

  Step 3：验证锁定（次日+3天后各验证一次）
    → 连续2次做对 → 标记"攻克（待巩固）"
    → 连续3次做对 → 标记"已攻克"

  Step 4：写入档案
    更新该条目状态 + 设置验证提醒

攻克后复发处理：
  → 攻克后再次出现同子类型错误 → 降级为"突破中"
  → 连续2次复发 → 重新启动专项突破流程

月报统计维度：
  → 五维度占比饼图
  → 各维度内子类型占比排序
  → 图景建立错误占比（物理独有指标）
  → 顽固弱项清单（含出现次数+当前状态）
  → 跨维度交叉分析（如有）
```

## 九、与通用错题本的协作边界

### 9.1 职责分工

```text
通用错题本职责（表面层）：
  → 接收所有科目错题（统一入口）
  → 拍题三信息法收集信息
  → 基础四维分类（概念/计算/审题/策略）
  → 记录错题表面信息
  → 跨科目统一管理与学期报告汇总
  → IM提醒统一调度

物理错误DNA职责（深度层，本维度表）：
  → 物理专属五维度深度分类
  → 子类型精确定位（P01/C03/F02/R01/T05等）
  → 跨维度关联分析
  → 顽固弱项专项突破（四步流程）
  → 物理焦虑处理
  → 物理弱项月报生成
```

### 9.2 维度映射表

通用错题本的基础四维分类与本维度表的深度分类映射关系：

```text
通用错题本四维标签    →  本维度表可能映射       →  判断依据
───────────────────────────────────────────────────────────
概念理解错误          →  P类(图景)或C类(概念)   →  是否涉及图景缺失？
                       P类：因为没画图所以"不理解"
                       C类：概念本身理解偏差
计算/操作失误         →  T类(数学工具)或F类     →  物理思路对吗？
                       T类：物理思路对但数学算错
                       F类：公式用错导致计算方向错
审题习惯问题          →  P类(图景)或R类(过程)   →  是否图景/过程遗漏？
                       P类：漏条件因为没画图
                       R类：条件都有但过程分析不完整
策略选择错误          →  R类(过程)或F类(公式)   →  方法选择还是过程划分？
                       R类：过程划分策略错
                       F类：公式选择策略错
```

交接时，通用错题本将基础四维标签作为 `physicsBasicDimension` 字段推送，本SKILL据此进入对应维度的子类型精确定位。由于通用四维→物理五维映射不是1:1，本SKILL会结合题目类型标签（力学/电学/光学/热学）进行综合判断。

### 9.3 数据流转规则

```text
交接流程：
  通用错题本完成基础分类 → 推送交接记录至物理错误DNA → 本维度表进行深度分类
  物理错误DNA完成深度分析 → 回写至通用错题本的"物理专属字段"区域
  不产生重复记录：通用层记表面，物理层记根因

交接记录格式：
  {
    "source": "correction-notebook",
    "type": "新错题 / 顽固弱项确认 / 深度分析请求 / 焦虑信号",
    "physicsBasicDimension": "图景建立错误 / 概念混淆 / 公式误用 / 过程分析错误 / 数学工具错误",
    "surfaceInfo": {
      "题目摘要": "...",
      "学生答案": "...",
      "正确答案": "...",
      "表面根因": "...",
      "题目类型标签": "力学/电学/光学/热学"
    },
    "historyRefs": ["错题ID1", "错题ID2"],
    "timestamp": "..."
  }

回写字段：
  物理深度分类：已交接至基因档案
  物理子类型ID：如 P01/C03/F02/R01/T05
  物理跨维度关联：如 "主维度P01+次要维度C01"
  物理根因分析（深度）：一句话深度根因
  物理弱项状态：无 / 观察 / 顽固 / 突破中 / 已攻克
```

### 9.4 触发去重规则

```text
物理科目3次同类错误：
  → 通用错题本：仅标记"待物理错误DNA处理"，不独立启动预警
  → 物理错误DNA：执行顽固弱项四步突破流程

物理科目固定错误模式：
  → 通用错题本：不独立生成递进练习
  → 物理错误DNA：执行四步突破流程

物理焦虑信号词：
  → 通用错题本：检测后转交物理错误DNA
  → 物理错误DNA：执行"数据替代情绪+图景重建"流程

IM提醒：
  → 物理错误DNA：指定提醒需求（何时提醒、提醒内容）
  → 通用错题本：统一调度，避免多个SKILL同时提醒
```
