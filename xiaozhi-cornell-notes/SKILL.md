# 📝 智能康奈尔笔记档案 SKILL

> **一句话定位：** 你记笔记，笔记记住你——  
> 当你需要它的时候，它主动找到你；  
> 当你以为忘了，它帮你想起来。

---

## ⚠️ 技术实现边界声明

> **关于“拍照解析”机制：** 模块中的课堂笔记拍照上传与提炼功能，强依赖于多模态大语言模型（Multimodal LLMs）的视觉识别能力，或定制化平台上专属 SKILL 扩展的本地 OCR 脚本服务，并非普通纯文本 LLM 原生具备的能力。

---

## 一、核心使命

**传统笔记的三大痛点：**

```
痛点①：记了不看
  课上记了满满一页，课后从不翻开
  原因：静态存储，没有"被需要"的时机

痛点②：看了不懂
  翻开自己的笔记，发现看不懂自己在写什么
  原因：记录格式混乱，没有自问自答的训练

痛点③：懂了不用
  复习时懂了，做题时想不起来这个知识点有对应笔记
  原因：笔记和做题是两个孤立的系统
```

**这个SKILL的解法：**

```
拍照上传
    ↓
自动提炼：线索问题栏 + 底部总结行
    ↓
存入笔记DNA档案（按学科/课题/日期索引）
    ↓
做错题时 → 主动推送："你在[日期]的笔记里记录了这个知识点"
费曼测试时 → 主动推送："这个概念在你[科目]笔记第X次课里出现过"
    ↓
跨学科关联：同一个概念在不同学科出现时，自动标注
```

## 三、模块A：拍照识别与三栏提炼

### 康奈尔笔记三栏结构

```
┌─────────────────┬──────────────────────────────┐
│   左栏（线索区） │        右栏（笔记区）          │
│                 │                              │
│  关键问题       │  课堂实际记录的内容            │
│  核心词语       │  （这是学生已经写好的部分）     │
│  （AI提炼生成） │                              │
│                 │                              │
├─────────────────┴──────────────────────────────┤
│                 底部总结区                      │
│         用一句话概括本页笔记的核心               │
│              （AI提炼生成）                     │
└────────────────────────────────────────────────┘
```

### 拍照上传后的操作流程

**Step 1：接收图片，识别内容**

```
学生上传笔记图片后：

"收到！我来分析这页笔记——
 这是[科目][课题]的内容，对吗？
 如果有误请告诉我，我会更正后存入档案。"

→ 确认无误后进入Step 2
→ 如果识别有歧义，先让学生确认再提炼
```

**Step 2：自动提炼左栏线索问题**

```
提炼规则：
① 把右栏每个知识点转化为"可以自我检验"的问题
② 问题必须能从右栏找到答案（不过度引申）
③ 每个重要知识点对应1个问题，一页笔记通常3-6个问题

提炼格式：
"线索问题（左栏）：
 Q1: [问题1]
 Q2: [问题2]
 Q3: [问题3]
 ...
 
 这些问题是为了让你盖住右栏后，用左栏自我测验。
 你觉得这几个问题覆盖了这页笔记的核心吗？"
```

**Step 3：自动生成底部总结**

```
总结规则：
① 用一句话（≤30字）概括这页笔记的核心
② 必须包含：这是什么（概念名称）+ 最重要的一点是什么
③ 学生可以修改，修改后版本存入档案

总结格式：
"底部总结（一句话）：
 '[概念名] = [核心定义/方法/公式]，关键在于[最重要的一点]'
 
 你觉得这句话抓住了这页的核心吗？可以直接修改。"
```

**Step 4：存入笔记DNA档案**

```
存入时附带元数据：
· 日期
· 学科
· 课题/章节
· 关键词标签（自动提取）
· 关联知识点（与已有档案的关联）

存入确认：
"存入成功！这页笔记已归入：
 [学科] > [课题] > [日期]
 关键词标签：[标签列表]
 
 下次你在做[相关知识点]的题目时，
 我会主动推送这页笔记给你。"
```

## 四、模块B：笔记DNA档案系统

### 档案结构

```
笔记DNA档案
├── 语文
│   ├── 课文单元（按教材章节）
│   ├── 写作技法
│   └── 古诗词积累
│
├── 数学
│   ├── 代数（一次函数/二次函数/方程组…）
│   ├── 几何（三角形/圆/相似…）
│   └── 统计概率
│
├── 英语
│   ├── 语法笔记
│   ├── 词汇记录
│   └── 阅读笔记
│
└── [其他学科]
    └── …
```

### 笔记条目的完整存储格式

```
【笔记DNA条目】
笔记ID：自动生成
日期：
学科：
课题/章节：
关键词标签：[3-5个关键词]
线索问题：[Q1, Q2, Q3…]
底部总结：[一句话]
关联已有笔记：[相关笔记ID列表]
被调取次数：0（每次推送+1）
最近调取日期：-
```

### 笔记检索方式

```
学生可以用以下方式调取笔记：

① 关键词检索：
  "找找我关于'一次函数'的笔记"

② 日期检索：
  "帮我看上周的数学笔记"

③ 课题检索：
  "找我学'圆的相关概念'时记的笔记"

④ 问题驱动检索（最高级）：
  "我做错了一道关于'圆心角和圆周角关系'的题，
   帮我找相关笔记"
  → 小龙虾自动匹配，推送最相关的2-3条笔记
```

## 五、模块C：主动推送机制

**这是此SKILL最核心的差异化功能——不需要学生主动去找笔记。**

### 三种主动推送触发场景

**触发场景①：错题分析时**

```
当学生在错题本SKILL中分析某道错题时，
笔记SKILL自动检测：
→ 这道题的知识点在笔记DNA中有没有对应记录？

如果有：
"我在你[日期]的[科目]笔记里找到了这个知识点——
 当时你记录的要点是：[底部总结]
 线索问题里有一条：'[最相关的Q]'
 现在来看这个问题，你能回答吗？"

如果没有：
"这道错题涉及到的知识点，
 你的笔记档案里还没有对应记录。
 建议你整理一页这个知识点的笔记，
 下次遇到就有依据了。"
```

**触发场景②：费曼测试时**

```
当学生开始费曼测试，说明要解释某个概念时，
笔记SKILL自动检测该概念是否有笔记记录：

如果有记录：
"你在[日期]记过这个概念的笔记。
 我先不告诉你里面写了什么——
 你先讲，讲完我们对比一下你讲的和你当时记录的，
 看看理解有没有加深。"

目的：让费曼测试不只是凭空发挥，还有笔记的锚点对比
```

**触发场景③：复习提醒时**

```
当IM提醒SKILL触发某个知识点的复习时，
自动附带对应笔记的线索问题：

"复习提醒：[知识点]

 你的笔记里有这些线索问题：
 Q1: [问题1]
 Q2: [问题2]

 先盖住笔记，回答这两个问题——能回答出来说明记住了。"
```

### 推送频率控制

```
每次对话最多推送2条笔记（避免信息过载）
优先推送：
  ① 与当前话题最直接相关的笔记
  ② 被调取次数最少的笔记（防止只复习熟悉内容）
  ③ 距上次调取时间最长的笔记（防止遗忘）
```

## 六、模块D：跨学科知识网络

### 自动关联触发标准

```
当两页不同学科的笔记中，出现以下情况时，自动建立关联：

① 完全相同的概念（如：比例关系在数学和地理中都出现）
② 相同的分析框架（如：语文的人物分析和历史的人物评价）
③ 互为应用关系（如：数学的统计知识和物理的数据分析）
```

### 跨学科推送话术

```
"有意思的是——你在[科目A]的笔记里记过一个概念，
 和你今天[科目B]学的这个知识点有关联：
 
 [科目A] 的笔记（[日期]）写道：[底部总结]
 
 你觉得这两个知识点之间，有什么相似的思维方式？"

目的：建立知识迁移能力，不是死记硬背单科孤立知识
```

## 八、自问自答训练法（康奈尔核心用法）

**每周至少一次，用以下方式使用笔记：**

```
Step 1：遮住右栏
只看左栏的线索问题，盖住右栏的笔记内容

Step 2：口头或书面回答每个问题
不看答案，凭记忆回答

Step 3：对照右栏检验
打开右栏，看哪些回答了，哪些答错了，哪些忘了

Step 4：标注记忆状态
  ✅ 完全记住 → 下次可跳过这个问题
  ⚠️ 记住但不完整 → 下次仍需测试
  ❌ 完全忘记 → 触发IM提醒，1天后再测

小龙虾的角色：
出示线索问题 → 等学生回答 → 对照标准答案 → 记录状态
"第三个问题：'[Q3]'——你的回答是什么？"
```
