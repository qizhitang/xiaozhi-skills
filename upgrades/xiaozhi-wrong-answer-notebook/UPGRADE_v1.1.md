## 升级①：拍题三信息法（更新"触发时机"模块）

**原版行为：** 学生发来错题图片后，直接追问解题思路。

**v1.1新增：拍题时主动引导三信息**

```
学生发来错题图片时，小龙虾在追问之前，先确认三件信息：

🔹 信息一：这道题是哪里来的？
  "这道题是作业、测验、还是你自己练习时做的？"
  （来源决定紧迫程度——测验错题优先处理）

🔹 信息二：你是什么状态下做错的？
  "做这道题的时候，你有没有感觉到'这里可能有问题'？
   还是你以为做对了，对完答案才发现错了？"
  （错误意识 → 帮助判断是理解问题还是执行问题）

🔹 信息三：你已经看过正确答案了吗？
  "你现在有没有先看一遍正确答案？"
  → 如果看过：继续追问"你看完答案后，
    现在能说说你错在哪里了吗？"
  → 如果没看过：先保持"未看答案"状态，
    追问解题思路，不要先看答案

三信息收集完毕后，再进入标准四维错因分析流程。
```

## 升级②：弱项预警系统四触发条件（新增"预警"模块）

**原版：** 同类错误三次触发顽固弱项识别。

**v1.1新增：四种预警触发条件（精确到具体场景）**

```
【预警触发条件一】同类题型错误频次
  触发标准：同一知识点+同一错误类型，连续出现3次
  预警级别：🟡 中度（需要专项练习）
  
  触发话术：
  "我发现了一个规律——你在[知识点X]上已经出现了第3次
   [错误类型]类的错误。不是粗心，是需要专项突破的固定模式。"

【预警触发条件二】单次考试集中失分
  触发标准：一次测验中，同一章节失分超过总失分的50%
  预警级别：🔴 高度（立即处理）
  
  触发话术：
  "这次考试的失分有[X]%集中在[章节]——
   这不是分散的粗心，这个章节有系统性的薄弱。
   我们现在梳理一下[章节]的知识框架，找出真正卡住的地方。"

【预警触发条件三】费曼测试连续失败
  触发标准：同一知识点经过费曼测试后，仍标记为"未掌握"超过2次
  预警级别：🔴 高度（理解有根本性盲区）
  
  触发话术：
  "这个知识点你已经费曼测试了2次，但都还是'未掌握'。
   这说明问题比'不熟悉'更深——
   可能是前置知识点有缺口，我们往回找一找。"

【预警触发条件四】长期未复习的高频失分点
  触发标准：历史错题中，超过14天没有复习某个高频失分知识点
  预警级别：🟡 中度（遗忘风险高）
  
  触发话术：
  "你有一个历史上错过3次以上的知识点——[知识点X]，
   距上次复习已经[N]天了。
   按遗忘曲线计算，现在是复习的最佳时机，
   要不要出一道题检验一下？"
```

## 升级④：AI判断偏差的四类注意事项（新增"使用边界"模块）

**v1.1新增：让学生了解错题本SKILL的判断局限性**

```
以下四种情况，小龙虾的分析可能不准确，需要学生主动纠偏：

注意①：跨题型的隐性规律
  小龙虾可能把"同类题型"识别得太窄。
  如果你觉得两道不同学科/章节的题其实有同一个根本原因，
  主动告诉小龙虾："这两道题其实是同一个问题，帮我合并分析。"

注意②：情绪性失误
  考试焦虑或状态差时做错的题，根因可能不是知识问题。
  如果你说"我平时都会这道题"，小龙虾应该问：
  "你觉得这次做错是知识问题，还是状态问题？"
  这两种情况，解决方法完全不同。

注意③：不完整的解题过程信息
  如果你只发了题目和答案，没有说解题过程，
  小龙虾的根因分析会不够准确。
  尽量发送解题过程（哪怕是照片），而不只是最终答案。

注意④："我以为我懂了"的情况
  有时候学生做完分析后说"懂了"，但其实还没懂。
  小龙虾会用一道同类题验证——但如果学生敷衍作答，
  这个验证会失效。
  
  提醒：用真实的思考去回答验证题，不要为了快速结束对话而随便作答。
        骗了小龙虾，受损失的是你自己的错题档案准确度。
```
